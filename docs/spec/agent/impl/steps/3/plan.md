## 2. このフェーズのゴール

- `plan.md` の新規タスクに従って、対象ソースコードを自律的に書き換えることができること。
- `yarn tsc` (型チェック) および自動テストを実行し、その合否をパースできること。
- 検証失敗時に、エラー内容をプロンプトにフィードバックして最大3回まで自己修復を繰り返し、それでも失敗した場合は直前のクリーンな状態へ自動ロールバックできること。

## 5. 各タスク詳細

### T1: `runner.ts` の作成
- **ファイル**: `.agents/skills/flow/scripts/runner.ts`（新規）
- **内容**:
  * `yarn typecheck` や `yarn test` などのコマンドを実行する。
  * 実行結果の終了コードと、エラーログ（stdout/stderr）をキャプチャし、JSON 形式で出力する。
- **検証**:
  * 意図的に型エラーのあるファイルを作成し、スクリプトを実行してエラー情報が正しく取得できることを確認する。

### T2: `rollback.ts` の作成
- **ファイル**: `.agents/skills/flow/scripts/rollback.ts`（新規）
- **内容**:
  * `backup()`: パイプライン開始時に未コミットの変更がある場合、`git stash` または一時ファイルにバックアップを作成する。
  * `restore()`: 自己修復が失敗した際に、`git reset --hard` や `git checkout --` で元の状態に戻し、バックアップがあれば復元する。
- **検証**:
  * スクリプトを実行し、テスト用に作成した変更が正しくロールバックされることを確認。

### T3: `step4-impl-sync.md` の作成
- **ファイル**: `.agents/skills/flow/skills/step4-impl-sync.md`（新規）
- **内容**:
  * 計画書 `plan.md` に追加された新規タスク（`T-ADD-N` など）を元に、ソースコード（`src/client/`, `src/server/` など）を実際に書き換える手順。
  * 書き換え後、`runner.ts` で検証を実行し、エラーがあればそのエラーログを元に修正を最大3回試みるループロジック。
  * 3回失敗した場合は `rollback.ts` を使ってロールバックを実行する手順。
- **検証**:
  * テストタスクに対してエージェントが自律的に動き、コード修正・型チェック・テスト・修復ループが正常に完了すること。
