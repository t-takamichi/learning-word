# 実装フェーズ定義: 自律型開発オーケストレーター (flow)

本ドキュメントは、`/flow` コマンド（開発オーケストレーター）を自律的に構築するための実装レイヤーと各フェーズの分割を定義します。

## 1. 実装レイヤーの方針

本機能はエージェント環境（メタ開発機能）であるため、以下のレイヤー構造で実装を進めます。

1. **インフラ・エントリーレイヤー (Command & Entrypoint)**
   * エージェントの起動用インターフェース（`.agents/skills/flow/SKILL.md` の定義）
   * 統合コントローラーおよび状態管理オブジェクトの開発
2. **アナライザー・パーサーレイヤー (Diff & Impact Analyzer)**
   * `git diff` やファイルシステムの変更を検出・解析するヘルパースクリプト
   * 設計変更から影響を受ける計画書（`plan.md`）の特定アルゴリズム
3. **エグゼキューターレイヤー (Task Executor & Self-Repair)**
   * サブエージェントや他スキルの呼び出し、プロンプトの連結実行
   * ビルド（`tsc`）やテストコマンドの自動実行、エラーパース、自己修復ループ制御
   * ロールバック制御（Gitコマンド連携）

---

## 2. フェーズ分割一覧

| フェーズ | タイトル | 目的・ゴール | 依存フェーズ |
|---------|----------|------------|------------|
| **Phase 1** | コマンド基盤と設計自動更新 | `/flow` の起動、ペルソナ確立、ユーザー指示から設計書の自動更新ができる。 | なし |
| **Phase 2** | 計画書（plan.md）の自動同期 | 設計書の差分を検出し、影響を受ける実装計画書を自動で更新・マージできる。 | Phase 1 |
| **Phase 3** | 自動実装と自己修復・ロールバック | 計画書からコードを自動修正し、型チェックでエラーが出たら自己修復し、ダメならロールバックできる。 | Phase 2 |
| **Phase 4** | 逆同期と全体のE2E検証 | `/flow --reverse`（コードから設計への逆反映）を実装し、全体の統合テストを完了する。 | Phase 3 |
