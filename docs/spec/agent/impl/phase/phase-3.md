# Phase 3: 自動実装と自己修復・ロールバック

## ゴール（このフェーズ完了時に動くもの）

- `plan.md` の新規タスクに従って、対象ソースコードを自律的に書き換えることができる
- `yarn tsc` (型チェック) および自動テストを実行し、その合否をパースできる
- 検証失敗時に、エラー内容をプロンプトにフィードバックして最大3回まで自己修復を繰り返し、それでも失敗した場合は直前のクリーンな状態へ自動ロールバックできる

---

- [オーケストレーター設計書](../../orchestrator.md) (機能要件 AC4, AC5, AC6)

## 完了条件（Done of 定義）

- [ ] 新しいタスクを追加した際、対象の React/Hono などのコードが自動で修正されること
- [ ] 修正後に `yarn tsc` や自動テストが自動的に実行されること
- [ ] エラー検出時、自動的に最大3回の再修正（修復ループ）が走り、ログに「自己修復を試みています...」と表示されること
- [ ] 修復が最終的に失敗した際、`git reset --hard` や `git checkout` がトリガーされ、対象ファイルが起動前の状態に復元されること
- [ ] ロールバック後に「修復失敗のため、ロールバックを実行しました」とユーザーへ詳細ログが通知されること

## 想定リスク

| リスク | 優先度 | 対策 |
|--------|--------|------|
| **R3: 自動ロールバックにより、ユーザーの未コミットな手動コード変更が消失する** | 高×高 | `/flow` のパイプライン開始時に必ず `git status` を確認する。未コミットの変更がある場合、自動的に `git stash save "flow-pre-backup"` を作成して手動変更を退避させてから実行する。あるいは、起動時に「未コミットの変更があります。コミットするか、退避させてから再実行してください」と警告を出して処理を中断する。 |
