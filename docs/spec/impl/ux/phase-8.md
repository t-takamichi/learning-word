  * `yarn db:reset` コマンドでいつでも綺麗にデータベースをクリーン＆シードできる環境を整備する。

### 📦 T-8.6: ユーザーセキュリティの強化 (PIN認証と一覧廃止・認可保護)
* [ ] **共通の認証ミドルウェア (Hono authMiddleware) の導入**: `X-User-Token` からユーザーを特定し、`c.get('user')` にユーザー情報を保存するミドルウェアを作成。
* [ ] **データ系ルートのトークン認可保護**: `/api/session`, `/api/review`, `/api/words` でクライアントから `userId` パラメータを受け取らず、ミドルウェアで解決された `userId` を適用するよう変更。
* [ ] **PBKDF2の暗号強度向上 & ログイン攻撃対策**:
  * PBKDF2のイテレーション回数を **210,000回** に変更。
  * PINの検証において `crypto.timingSafeEqual` を用いた定数時間比較を実装。
  * ログインAPI (`POST /api/users/login`) に、偽装不可能な接続元IP（connInfo）およびユーザー名単位（グローバル）の失敗カウントに基づくレートリミットを導入（総当たり攻撃および setInterval による定期クリーンアップによるメモリリーク防止）。
  * ユーザー登録API (`POST /api/users`) で名前重複時のエラーメッセージを汎用的なもの（列挙攻撃防止）に変更。
* [ ] **ユーザー削除時の本人確認（PIN検証）の導入**:
  * `DELETE /api/users/:id` 実行時に、リクエストボディで合言葉（`pin`）を受け取り、本人の `pin_hash` と一致しているかを検証する。
  * PINが未入力の場合は `400 Bad Request`、PINが一致しない場合は `403 Forbidden` を返す（`401` はトークン認証の失敗＝自動ログアウト対象に限定し、意味を分離する）。
* [ ] **マイグレーションの既存ユーザー無効化 & スキーマ整合**:
  * `db.ts` の移行処理において、既存のユーザーデータをクリア（物理削除）し、新規に再登録を促す。
  * 移行後のDBでもインデックス等が一貫するよう schema と migration を調整。
* [ ] **フロントエンド hook (`useUsers.ts`) の不要ダミー掃除**: 旧 `users: []` 等の不要になったダミープロパティやメソッドを完全に削除・クリーンアップ。
* [ ] **フロントエンドの API 呼び出し・認可対応**: クライアント側からリクエスト時の `userId` 送信を削除し、代わりに `X-User-Token` ヘッダーを自動付与して送信するように変更。

## 3. 受け入れ基準 (Acceptance Criteria)

* **複数ユーザー機能**:
  * [ ] ユーザー選択後（ログイン成功後）、ヘッダーに選択したユーザー名とアバターが表示されること。
  * [ ] ユーザーを切り替えた際、学習進捗（weakやstreak等）がそのユーザー固有のものに切り替わり、他のユーザーに干渉しないこと。
  * [ ] 新規登録時、名前が空または重複している場合に、やさしい応援形のコピーでエラーが返されること。
  * [ ] 全ユーザーの一覧を表示する画面/APIが廃止され、ログインには「名前＋PIN/合言葉」が必要なこと。
  * [ ] `DELETE /api/users/:id` は、トークン無効時は 401、他人のユーザー指定時は 403 を返し、本人のみ削除可能なこと。さらに削除時の合言葉（PIN）が未入力なら 400、不一致なら 403 を返すこと。
* **レベル選択機能**:
  * [ ] 学習開始前にレベル（初級/中級/上級）が選択でき、選択したレベル内の単語のみが出題されること。
  * [ ] 各レベルカードに進捗率（〇〇/〇〇語）とプログレスバーが正確に表示されること。
  * [ ] 100%完了時に、制覇マーク（星/王冠）が表示されること。
* **データ管理 & テスト容易性 (Attention to Detail)**:
  * [ ] `yarn db:reset` を実行した際、データベース（learning.db）が初期化され、`db/seed.json` に定義された 3 つのレベル（Basic, Intermediate, Advanced）の単語セットが正しく投入されること。
* **ディズニー品質**:
  * [ ] ユーザー選択成功時、マスコットが喜ぶアニメーションとウェルカム音が遅延なく（200ms以内）再生されること。
  * [ ] 画面レイアウト（特にカードやボタン）のタップターゲットがすべて 44px 以上確保されていること。
