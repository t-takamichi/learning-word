# Phase 10: 登録上限・セット単語数表示（Could have）

## ゴール（このフェーズ完了時に動くもの）
- 1ユーザーあたりの単語・単語セットの登録件数に上限が設けられ、濫用による無制限のデータ増加を防止する。
- 単語セット一覧に、そのセットに含まれる単語数（自分から見える件数）が表示される。

## 対応する設計
- [一般ユーザーによる単語・単語セット登録](../../design/user-word-management.md)（Could have）

## スコープ（このフェーズでやること）

| 層 | 実装項目 |
|----|---------|
| Domain | 上限値の定数定義、上限超過エラーの型 |
| Infrastructure | `userWordRepository` / `userWordSetRepository` に、対象ユーザーの現在件数を数える `countByUser(userId)` を追加 |
| Application | `CreateUserWordUseCase` / `CreateUserWordSetUseCase` 内で作成前に件数を確認し、上限超過時はエラーを返す |
| Presentation | 上限超過時に400（または429）でエラーメッセージを返す |
| Frontend | ・**単語数バッジ**: 単語セット一覧等で表示する単語数カウンターは、`--lavender-500` または `--berry-100` を背景にした丸っこいピルチップで表現する<br>・**上限エラー表示**: 上限到達時のエラーは、機械的な拒絶ではなく「いまはここまで！よくがんばったね。新しく追加するまえに、いまの単語をたくさん復習してあげよう🌸」というやさしい労いのマイクロコピーで表示する |

## スコープ外（このフェーズでやらないこと）
- 上限値の管理画面からの変更（環境変数や定数でのハードコードで十分とする）

## 依存フェーズ
- Phase 7（単語CRUD）
- Phase 8（単語セットCRUD）

## 完了条件（Doneの定義）
- [ ] 上限件数を超えて単語・単語セットを作成しようとするとエラーになり、作成されないこと
- [ ] 単語セット一覧に、自分から見える単語数が表示されること
