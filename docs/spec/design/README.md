# 設計ドキュメント: 複数ユーザー対応 & レベル別単語セット選択

## 1. プロダクト背景と目的 (STEP 1)

### ユーザーストーリー

* **複数ユーザー対応**
  * **As a** 複数の学習者（例: 家族、同僚、複数の個人アカウントなど）、
  * **I want to** 自分専用のユーザーアカウント（プロファイル）を作成・選択して学習を記録したい、
  * **So that** 他人の学習進捗（weakやmasteredなどの判定）に干渉されることなく、自分自身のペースで苦手な単語を効率的に学習したい。
* **レベル別単語セット選択**
  * **As a** さまざまな英語力を持つ学習者、
  * **I want to** 自分のレベル（初級、中級、上級など）に合った単語セットを明示的に選択して学習したい、
  * **So that** 自分の現在のスキルレベルに合わない難しすぎる（または簡単すぎる）単語で時間を浪費するのを防ぎ、適切な難易度で語彙力を高めたい。

### ターゲットユーザー（ペルソナ）

1. **言語学習者（一般）**
   * 自主的に英語を学習している個人。スマートフォン（主にモバイルブラウザ）で通勤時間やスキマ時間に学習を行う。
   * 技術リテラシー：中程度（一般的なスマホアプリの操作ができるレベル）。
   * 場面：毎日のスキマ時間（3分〜5分程度）にオートプレイやクイズ形式でサクッと学習したい。
2. **共有デバイスでの学習者（例：家族、教育環境）**
   * 同一のPCやiPad等の端末を複数人で共有して学習を進める環境。
   * 技術リテラシー：子供から大人まで多様。

### 解決するペインポイント

* **記録が混ざる問題**
  * 現状、単一のSQLiteデータベースでグローバルに学習進捗を保存しているため、1つの端末やサーバーで複数人が学習しようとすると、学習進捗（どの単語が弱点か）が混ざり合い、パーソナライズされた弱点克服機能が機能しなくなってしまう。
* **単語の難易度が固定されている問題**
  * アプリ起動時にプリセットされた全単語がランダムに出題されるため、「すでに知っている簡単な単語」ばかりが出たり、逆に「まだ学習していない専門的な難解単語」が混ざったりして、自分のレベルに合わせた段階的な学習ができない。

### 成功指標 (KPI)

* **ユーザー切り替えの正確性とセッション完了率**:
  * ユーザー選択後に、該当ユーザーの弱点単語（`weak` 判定された単語）が学習セッションに正しく 4 件ブレンドされ、学習セッション（10語のクイズ）を途中で離脱せずに最後までやり遂げた比率（セッション完了率 90% 以上を維持）。
* **レベル選択の利用率**:
  * 学習開始時に、デフォルトの「basic」以外の単語セット（中級・上級など）が全体の学習セッションの 40% 以上で選択され、自分の語彙力に合ったパーソナライズ学習が実行されていること。
