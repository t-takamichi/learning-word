# サウンド・ハプティクス仕様 (Sound & Haptics)

本ドキュメントは、モバイル（特に iOS Safari）におけるブラウザの厳しい自動再生制限やマナーモード仕様を考慮し、遅延なく確実に動作する効果音とハプティクス（振動）の設計を定義します。

---

## 1. コンセプト：「音が鳴らなくても、体験は100%成立する」

サウンドと振動は、学習体験を豊かにする「加点要素（Delight）」として位置づけます。
ユーザーがマナーモード（サイレントスイッチON）で操作している場合や、Android/iOSで振動が制限されている場合でも、視覚的なフィードバック（スパークル、トースト、マスコットの表情）によって、学習の達成感や状況が100%理解できるように二重設計を行います。

---

## 2. 効果音 (Sound Effects) 定義

軽量化と遅延防止のため、重いオーディオファイル（.mp3 / .wav）のダウンロードは行わず、**Web Audio API (OscillatorNode / GainNode)** を用いたリアルタイムな周波数合成（シンセサイズ）音源を採用します。これにより、ロード遅延「0ms」で瞬時に発音します。

| サウンドID | トリガーの瞬間 | 音質定義（合成仕様） | 長さ / ニュアンス |
|:---|:---|:---|:---|
| `correct` (Nice!) | 回答で「Good」を選択した瞬間。 | 三角波 (Triangle) / C6 (1046.5Hz) → E6 (1318.5Hz)。滑らかなポルタメント。 | 約0.2秒。明るく跳ねるようなピコーン音。 |
| `combo` | 5連続正解（コンボ）など、連続正解が一定数に達した瞬間。 | 三角波 / E6 (1318.5Hz) → G6 (1568Hz) → C7 (2093Hz) の上昇アルペジオ。 | 約0.3秒。テンポ良く上昇し、高揚感を与える。 |
| `again` | 回答で「Again」を選択した瞬間。 | サイン波 (Sine) / A3 (220Hz) → C4 (261.6Hz) への低いスライド。 | 約0.22秒。優しく落ち着いた低刺激の音。責めない。 |
| `flip` | フラッシュカードをタップして「めくった」瞬間。 | サイン波 / 350Hz → 150Hz への急激な下降スライド（エンベロープ制御）。 | 約0.08秒。カードの風切り音を模した「シュッ」という音。 |
| `complete` | 単語セッション（全10語など）が完了し、リザルト画面に遷移した瞬間。 | 三角波でのアルペジオ（C5→E5→G5→C6）＋ サイン波による最終和音（Cコード）の1秒持続フェードアウト。 | 約1.2秒。上品で心地よいファンファーレ。 |
| `tap` | 一般的なボタンやタブをタップした瞬間。 | サイン波 / 900Hz の極小音。 | 約0.03秒。指先にタップ感を返すための非常に短い「クリック」音。 |

## 3. iOS Safari の制約と対応設計 (iOS Autoplay Policy)

iOS Safari をはじめとするモバイルブラウザでは、ユーザーの物理タップ操作（User Activation）がない状態での音声再生は自動的にブロックされます。これを回避するため、以下の実装仕様を定義します。

1.  **初回の「明示的アクティベーション」の要求**:
    *   学習開始画面、あるいはユーザー選択画面の「スタート」「はじめる」といった大きなボタンの `onClick` ハンドラ内で、同期的に `AudioContext.resume()` (または 1サンプルの無音バッファ再生) を呼び出し、ブラウザの AudioContext を「Running（活性化）」状態にアンロックします。
2.  **遅延防止対策 (Web Audio API の使用)**:
    *   `<audio>` タグでの再生は、ストリーミングやデコードによる数百ミリ秒の遅延が発生し、タップとのズレを感じさせるため使用しません。すべて Web Audio API による Oscillator 発音に統一します。
3.  **Vibration API のフォールバック**:
    *   `navigator.vibrate` は iOS Safari では動作しません。
    *   実装時は必ず `if (typeof navigator !== 'undefined' && navigator.vibrate)` の存在チェックを行い、例外をスローせず安全にスルー（サイレントフォールバック）する構造にします。iOS 端末に対しては、音と視覚（スパークル）のみで同様のフィードバック価値を担保します。

---

## 4. ミュート（消音）UI/UX仕様

*   **ミュートのデフォルト状態**:
    *   初回起動時の初期値は「音あり」としますが、ユーザーがいつでもワンタップで消音できるようにします。
*   **UIの配置**:
    *   学習画面のヘッダー部、およびユーザー選択画面の分かりやすい位置に `SoundToggle` ボタン（スピーカーマーク 🔊 / 🔇）を常時配置します。
*   **永続化**:
    *   ミュートのオン/オフ状態は、ブラウザの `localStorage` に即座に保存します。次回起動時や画面遷移時もその設定を完全に引き継ぎます。

---

## 5. ハプティクス（物理振動）パターン仕様

Vibration API に対応したデバイス（Android Chrome等）で、指先に心地よい触覚フィードバックを伝えます。

*   **`correct`（正解）**:
    *   パターン: `[15]` (15msの短い単発振動)
    *   意図: 指先に「チッ」という小気味良い感触を返し、正解の瞬間を際立たせる。
*   **`combo`（コンボ達成）**:
    *   パターン: `[10, 30, 10]` (10ms振動 → 30ms休み → 10ms振動)
    *   意図: タタッと軽いダブルタップ感を返し、リズムを刻む。
*   **`complete`（セッション完了）**:
    *   パターン: `[15, 40, 15, 40, 30]`
    *   意図: 完了のファンファーレと同期し、お祝いの鼓動を伝える。
