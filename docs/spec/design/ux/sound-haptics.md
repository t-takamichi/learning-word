# Sound & Haptics — 音と触覚のフィードバック

親: [README.md](./README.md)

「正解した時に Nice の音が流れないとダサい」への直接の回答。
**iPhone Safari の制約を織り込んだ、確実に鳴るサウンド設計** を定義する。

---

## 1. サウンド一覧

| ID | 名前 | トリガー | 長さ | キャラクター |
|----|------|---------|------|-------------|
| `sfx.correct` | **Nice!** | Good タップ | ~0.4s | 明るい上昇2〜3音（キラッ） |
| `sfx.combo` | コンボ | 3/5/10 連続到達 | ~0.6s | さらに高く弾む上昇音 |
| `sfx.again` | もういちど | Again タップ | ~0.25s | やわらかい低め単音（責めない） |
| `sfx.flip` | めくり | カード裏返し | ~0.12s | 軽いポップ（任意・控えめ） |
| `sfx.complete` | ファンファーレ | 完了画面 | ~1.2s | 達成の和音アルペジオ |
| `sfx.tap` | タップ | 主要ボタン | ~0.05s | かすかなクリック（任意） |

**設計方針**: 短く・明るく・うるさくない。同じ音の連打で不快にならないよう、音量控えめ＋わずかなピッチ揺らぎを許容。

---

## 2. iOS Safari 制約と対応（重要）

iPhone Safari では音・触覚に強い制約がある。**設計時に必ず織り込む。**

| 制約 | 内容 | 対応 |
|------|------|------|
| オーディオ解錠 | ユーザー操作イベント内でしか音を鳴らせない（自動再生不可） | 最初のタップ（「れんしゅうする」等）で `AudioContext` を `resume()` して解錠。以降のプログラム再生を可能にする |
| サイレントスイッチ | 端末のミュートスイッチで Web Audio が鳴らない場合がある | 音が鳴らなくても **視覚フィードバックだけで体験が完結** するよう二重化（音は"加点"） |
| `navigator.vibrate` 非対応 | iOS Safari は Vibration API 未対応 | ハプティクスは"できれば"。iOS では視覚・音で代替し、対応端末（Android等）でのみ振動 |
| 遅延 | `<audio>` 要素は初回に遅延が出やすい | **Web Audio API** で事前デコード＆低遅延再生を推奨 |

---

## 3. 実装方針

### 3.1 推奨：Web Audio API（合成 or 事前デコード）
- 短い効果音は **コード生成（オシレータ）** も可 → 音源ファイル不要・超軽量・低遅延
  - 例：`sfx.correct` = 2音の三角波を 0.1s ずつ（C6→E6）にエンベロープを掛ける
- こだわる場合は短い `.mp3/.m4a` を `decodeAudioData` で事前ロード
- 単一の `AudioContext` をアプリで共有し、初回ユーザー操作で `resume()`

### 3.2 サウンドマネージャの責務（`useSound` フック想定）
```
- unlock():           初回タップで AudioContext を解錠
- play(id):           指定SFXを低遅延で再生（ミュート時は何もしない）
- setMuted(bool):     ミュート状態を localStorage に保存
- isMuted():          現在の状態
```

### 3.3 ミュートUI
- ヘッダーに 🔊/🔈 トグル（`--radius-pill`）。既定ON。
- 状態は `localStorage("berry.sound.muted")` に保持。

---

## 4. ハプティクス（触覚）

| イベント | パターン（対応端末のみ） |
|---------|------------------------|
| 正解 | `navigator.vibrate?.(15)`（ごく短く） |
| コンボ達成 | `vibrate?.([10, 30, 10])` |
| 完了 | `vibrate?.([15, 40, 15, 40, 30])` |

- **必ず存在チェック**（`if (navigator.vibrate)`）してから呼ぶ。iOS では自然に無効化される想定。
- 触覚は"あれば嬉しい"レベル。無くても体験が成立するよう設計する。

---

## 5. アクセシビリティ / 配慮

- 音は既定ONだが即ミュート可能。ミュート時も学習は完全に成立する。
- `prefers-reduced-motion` は視覚アニメに作用。音とは独立に扱う（音だけ残せる）。
- 大音量を避け、控えめなデフォルト音量（例 0.3〜0.5）にする。

---

## 6. 受け入れ基準（AC）

- [ ] 最初のユーザー操作後、`sfx.correct` が **遅延なく**（体感即時で）鳴る
- [ ] Good で "Nice!" が鳴り、Again では責める音が鳴らない
- [ ] ミュート状態が再訪時も保持される
- [ ] 音が鳴らない環境（サイレント/未対応）でも、正解の実感が視覚だけで得られる
- [ ] iOS Safari で `navigator.vibrate` 呼び出しがエラーを起こさない（存在チェック済み）
- [ ] 完了画面で `sfx.complete` が1.2秒以内で鳴り終わる
