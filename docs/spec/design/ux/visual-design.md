# Visual Design System — "Berry" テーマ

親: [README.md](./README.md)

黒ベースから **ストロベリー・ピンク** ベースへ。単なる色替えではなく、
「やさしい・あまい・前向き」を体現するデザインシステムとして定義する。

---

## 1. カラーパレット

### 1.1 ブランドカラー

| トークン | HEX | 用途 |
|---------|-----|------|
| `--berry-500`（Primary） | `#FF6FA5` | メインのピンク。主要ボタン・アクティブ・ブランド |
| `--berry-400` | `#FF8FB8` | ホバー/明るいアクセント |
| `--berry-600` | `#F0568F` | 押下・強調 |
| `--berry-100` | `#FFE4EF` | 淡いピンク面・チップ背景 |
| `--berry-50` | `#FFF0F5` | 背景グラデ上端 |

### 1.2 アクセント（サブ）

| トークン | HEX | 用途 |
|---------|-----|------|
| `--mint-500`（正解カラー） | `#3FC7A0` | **正解（Good）・成功フィードバック**。ピンクと補色的に映える |
| `--mint-100` | `#DFF7EF` | 正解トースト・成功面の背景 |
| `--lavender-500` | `#B69CFF` | 補助アクセント（バッジ・XPゲージ） |
| `--sunny-400`（ごほうび） | `#FFC94D` | コンボ🔥・スター・お祝いの粒 |
| `--coral-500`（再挑戦） | `#FF7A6B` | Again（やさしい赤。警告色にしない） |

### 1.3 ニュートラル（真っ黒を使わない）

| トークン | HEX | 用途 |
|---------|-----|------|
| `--ink-900` | `#4A2C3A` | 見出し・強いテキスト（プラム寄りの濃色） |
| `--ink-700` | `#6E5561` | 本文 |
| `--ink-400` | `#A38C97` | 補助テキスト・プレースホルダ |
| `--surface` | `#FFFFFF` | カード・面 |
| `--surface-tint` | `#FFF7FA` | 面の微ピンク白 |
| `--bg-gradient` | `linear-gradient(180deg, #FFF0F5 0%, #FFE4EF 100%)` | 画面背景 |

### 1.4 状態カラー

| 状態 | 色 |
|------|-----|
| 成功 / 正解 | `--mint-500` |
| 再挑戦 / Again | `--coral-500` |
| 情報 | `--lavender-500` |
| エラー（管理画面の入力等） | `#E5484D`（ここだけ明確な赤を許容） |

### 1.5 黒→ピンク 置換対応表（現行CSSからの移行）

| 現行（黒ベース） | 新（Berry） |
|------------------|-------------|
| `background: #0c0d12` / `radial-gradient(#1c1d24→#0c0d12)` | `var(--bg-gradient)` |
| `color: #ffffff`（本文） | `var(--ink-700)` / 見出しは `--ink-900` |
| `rgba(255,255,255,0.04)`（カード面） | `var(--surface)` ＋ ピンク影 |
| `rgba(255,255,255,0.1)`（ボーダー） | `rgba(255,111,165,0.18)` |
| `#81c784`（正解の緑） | `var(--mint-500)` |
| 白グラデの英単語テキスト | `--ink-900`（可読性優先）＋ 任意で `--berry-500` の下線装飾 |
| `box-shadow: rgba(0,0,0,0.2)` | `0 8px 24px rgba(255,111,165,0.18)`（ピンクの影） |

> **注意**: ピンク面の上に白文字を置くとコントラストが不足しやすい。
> 主要ボタン（`--berry-500` 背景）の上の文字は **白（#FFFFFF）でAA以上**を満たすことを確認済みだが、
> 淡いピンク面（`--berry-100`）の上には **必ず `--ink-900` 系の濃色文字**を使う。

## 2. タイポグラフィ

丸みのある親しみやすい書体で「あまい」印象に。

| 用途 | サイズ | ウェイト | 色 |
|------|--------|---------|-----|
| 英単語（カード表面） | `clamp(2.4rem, 7vw, 3.4rem)` | 800 | `--ink-900` |
| 翻訳（カード裏面） | `clamp(1.5rem, 4.5vw, 2rem)` | 700 | `--mint-500`（=正解カラーと連動） |
| セクション見出し | 18–20px | 700 | `--ink-900` |
| 本文 | 16px | 500 | `--ink-700` |
| 補助・ヒント | 12–13px | 500 | `--ink-400` |

**フォントファミリー**:
```
font-family: "Hiragino Maru Gothic ProN", "Quicksand", "Nunito",
             system-ui, -apple-system, sans-serif;
```
- 日本語: ヒラギノ丸ゴ（iOS標準で丸ゴが出る）
- 英字: Quicksand / Nunito（丸ゴ系Webフォント。読み込めなければsystem-ui）
- 本文最小16px（iOS の入力ズーム防止）

## 3. 形・余白・影（Shape / Spacing / Elevation）

「かどをまるく、影をやわらかく」。

| トークン | 値 | 用途 |
|---------|-----|------|
| `--radius-card` | `28px` | カード |
| `--radius-btn` | `18px` | ボタン |
| `--radius-pill` | `9999px` | トグル・チップ・進捗 |
| `--space-unit` | `8px` | 余白の基本単位（8ptグリッド） |
| `--shadow-soft` | `0 8px 24px rgba(255,111,165,0.18)` | カード標準 |
| `--shadow-pop` | `0 12px 32px rgba(255,111,165,0.28)` | ホバー・浮き上がり |
| `--shadow-press` | `0 2px 8px rgba(255,111,165,0.20)` | 押下時 |

- ボタンは **ぷっくり感**（上ハイライト＋下ソフト影）。押すと少し沈む（`translateY(1px)` + `--shadow-press`）。
- 面は白ベースに `--surface-tint` の内側微グラデを許容。

## 4. マスコット（ブランドの相棒）

継続の鍵は「一緒にがんばる存在」。軽量なマスコットを導入する。

| 項目 | 仕様 |
|------|------|
| 名称（仮） | **Berry（ベリー）** — いちごモチーフの小さな相棒 |
| 実装方針 | まずは **絵文字/SVG 1点**（🍓 or 簡易SVG）で最小コスト導入。将来的に数パターンの表情差分 |
| 登場箇所 | ヘッダー隅・完了画面・空状態・正解時の小リアクション |
| 表情 | `idle`（通常）/ `happy`（正解）/ `cheer`（完了・ストリーク更新） |

> マスコットは"必須"ではなく **Should**。無くても成立するが、有ると継続率に効く（[quality-bar.md](./quality-bar.md) の感情曲線参照）。

## 5. 主要コンポーネント スタイル指針

### 5.1 フラッシュカード
- 面: `--surface`、角丸 `--radius-card`、影 `--shadow-soft`
- 表面の英単語は `--ink-900`。下に細いピンクの下線装飾（`--berry-200` 相当）を任意で
- 裏返し時: Y軸フリップ（180ms, ease-out）。`prefers-reduced-motion` 時はクロスフェード
- 「こたえを見る」ボタン: `--berry-100` 面 ＋ `--ink-900` 文字（アウトライン風）

### 5.2 評価ボタン（Again / Good）
| ボタン | 背景 | 文字 | 補足 |
|--------|------|------|------|
| **もういちど**（Again） | `--surface`（枠 `--coral-500`） | `--coral-500` | 失敗を責めない配色 |
| **できた！**（Good） | `--mint-500`（塗り） | `#FFFFFF` | 押すと正解演出＋"Nice!"音 |

### 5.3 進捗バー
- トラック: `--berry-100`、フィル: `linear-gradient(90deg, --berry-500, --sunny-400)`
- カウント（4/10）は `--berry-500` のピルチップ

### 5.4 単語リスト（希望.JPGの一覧UIを踏襲）
- 各行を白カード（角丸 `--radius-btn`、影 `--shadow-soft`）で分離
- 見出し語 `--ink-900`、翻訳 `--ink-700`、発音ボタンは `--berry-500` アイコン
- 学習ステータス（new/weak/mastered）を **小さなピンク/ミント/ラベンダーのドット**で表現

### 5.5 主要ボタン（CTA：例「れんしゅうする」）
- `--berry-500` 塗り＋白文字＋ぷっくり影。フル幅・角丸 `--radius-btn`・min-height 52px

### 5.6 自動再生コントロール（Auto-Play）の再配置（分析 F-9）
- 現状は秒数の数値入力を含む設定がヘッダー上部に常時露出し、学習の主役（カード）を食っている
- **既定は折りたたむ**：ヘッダーには小さな「じどうめくり」ピルトグルのみ。秒数設定は展開時に表示
- 展開UIも `--berry-100` 面＋`--ink-*` 文字でやわらかく。数値入力は最小限に
- 主役は常にカード。設定は"控えめな脇役"の視覚重みに落とす

### 5.7 学習ステータス色（ライト背景版）
現状の new/weak/mastered バッジ（暗背景前提の amber/red/mint）を、ライトなBerry面でも映える配色に置換。

| ステータス | 現状（暗背景） | 新（ライト背景・ドット/チップ） |
|-----------|--------------|------------------------------|
| new | `#fbbf24`（amber） | `--lavender-500`（やさしい新しさ） |
| weak | `#f87171`（red） | `--coral-500`（責めない要復習） |
| mastered | `#34d399`（mint） | `--mint-500`（達成＝正解カラーと連動） |

- リスト行では**小さなドット**（[component-structure.md](./component-structure.md) の `StatusDot`）で軽く示す。強い赤で"ダメ"を強調しない

## 6. CSS変数の実装位置（実装メモ）

`src/client/index.css` に `:root` としてトークンを定義し、各 `*.module.css` はハードコード色をトークン参照に置換する。

```css
:root {
  --berry-500:#FF6FA5; --berry-400:#FF8FB8; --berry-600:#F0568F;
  --berry-100:#FFE4EF; --berry-50:#FFF0F5;
  --mint-500:#3FC7A0; --mint-100:#DFF7EF;
  --lavender-500:#B69CFF; --sunny-400:#FFC94D; --coral-500:#FF7A6B;
  --ink-900:#4A2C3A; --ink-700:#6E5561; --ink-400:#A38C97;
  --surface:#FFFFFF; --surface-tint:#FFF7FA;
  --bg-gradient:linear-gradient(180deg,#FFF0F5 0%,#FFE4EF 100%);
  --radius-card:28px; --radius-btn:18px; --radius-pill:9999px;
  --shadow-soft:0 8px 24px rgba(255,111,165,.18);
  --shadow-pop:0 12px 32px rgba(255,111,165,.28);
  --shadow-press:0 2px 8px rgba(255,111,165,.20);
}
```
