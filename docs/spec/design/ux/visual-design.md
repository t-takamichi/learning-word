# デザイントークン仕様 (Visual Design Tokens)

本ドキュメントは、アプリ全体に統一された世界観（Berryテーマ）とアクセシビリティを適用するため、CSS変数として実装されるデザイントークンの詳細仕様を定義します。

---

## 1. カラーシステム (Color Palette & Mapping)

### 1.1. 真っ黒の完全排除とコントラスト比の保証
*   **真っ黒（#000000）の排除**: 強い黒は「あまい・やさしい」世界観を損ねるため、一切使用しません。メインのテキスト色には深みのあるプラムブラウンである `--ink-900` を使用します。
*   **ピンクの背景におけるテキスト色**: 
    *   ブランドメインカラーの `--berry-500` (`#FF6FA5`) に対する白文字 (`#FFFFFF`) は、コントラスト比が **2.2:1** と非常に低く、読みやすさに難があります。
    *   そのため、ピンク背景上のテキストには、コントラスト比 **5.2:1** (AAクリア) を満たす濃い目のプラムブラウン `--ink-900` を重ねるか、白文字を使用する場合は背景色を濃いピンクの `--berry-600` (`#F0568F`) 以上にする設計を徹底します。

### 1.2. カラーパレット定義

| トークン名 | HEX実値 | 主な役割 | コントラスト担保 |
|:---|:---|:---|:---|
| `--berry-500` | `#FF6FA5` | メインブランド。基本のインタラクティブ要素。 | 文字色は `--ink-900` とする |
| `--berry-400` | `#FF8FB8` | ホバー状態。やや明るめのピンク。 | 装飾目的のみ |
| `--berry-600` | `#F0568F` | アクティブ状態・白文字を重ねる場合の背景。 | 白文字とのコントラスト確保用 |
| `--berry-100` | `#FFE4EF` | 優しいピンクのバッジ背景・入力フィールドのベース。| テキストは `--berry-600` |
| `--berry-50`  | `#FFF0F5` | コンテナ背景などの極薄ピンク。 | テキストは `--ink-900` |
| `--mint-500`  | `#3FC7A0` | 正解・Goodボタン・初級タグ・完了のグリーン。 | 白文字との比 3.5:1 (太字) |
| `--mint-100`  | `#DFF7EF` | 正解バッジ等の薄い背景。 | テキストは `--mint-600` |

## 2. 黒・標準テーマから新テーマへの置換マッピング表

既存のCSSから、生値や標準的な色を排除し、Berryテーマのデザイントークンに置き換えるための対応ルールです。

| 既存の生値・標準指定 | 置き換え後のデザイントークン | 理由 / 体験上の意図 |
|:---|:---|:---|
| `#000000`, `black`, `#333333` | `var(--ink-900)` | 強い黒を排除し、柔らかく親しみやすいプラムブラウンに変更。 |
| `#666666`, `#888888` | `var(--ink-700)` | 本文や補助情報の読みやすさと、やさしいトーンの両立。 |
| `#E5484D` (強い赤) | `var(--coral-500)` | 「Again」や「エラー」で恐怖感を与えない、マイルドなコーラルレッド。 |
| `#00D293` (強い緑) | `var(--mint-500)` | 正解した時の気持ちよさと目にやさしいミントグリーン。 |
| `border-radius: 4px` / `8px` | `var(--radius-btn)` (`18px`) | UIパーツ（ボタン、入力欄）に「ぷっくり感」を出す丸み。 |
| `border-radius: 12px` / `16px` | `var(--radius-card)` (`28px`) | カードやダイアログの角を大きく丸め、安心感・おもちゃ感を与える。 |
| `box-shadow`（黒ベースの影） | `var(--shadow-soft)` | 影にブランドピンクをブレンドし、画面全体をあまいトーンにする。 |

## 3. タイポグラフィ (Typography)

モバイル Safari における拡大ズーム（16px未満の入力欄でズームが作動してしまう仕様）を防ぐため、**入力フォームのフォントサイズは最小16px**を厳守します。

*   **フォントファミリー (Font Family)**:
    *   日本語: `"Hiragino Maru Gothic ProN"` (ヒラギノ丸ゴ), `"Hiragino Kaku Gothic ProN"`, `sans-serif`
    *   欧文: `"Quicksand"`, `"Nunito"`, `system-ui`, `-apple-system`
    *   *丸みのある幾何学フォントと丸ゴシックの組み合わせで、あまい世界観を徹底します。*
*   **階層定義**:
    *   `Title (Large)`: `size: 24px` / `weight: 800` / `line-height: 1.3`
    *   `Title (Medium)`: `size: 18px` / `weight: 700` / `line-height: 1.4`
    *   `Body (Main / Input)`: `size: 16px` / `weight: 500` / `line-height: 1.5`
    *   `Body (Sub)`: `size: 14px` / `weight: 500` / `line-height: 1.5`
    *   `Hint / Date`: `size: 12px` / `weight: 700` / `line-height: 1.4`

---

## 4. 形状・余白・浮遊感 (Shape / Layout / Elevation)

*   **角丸 (Border Radius)**:
    *   `--radius-card`: `28px` (カード・モーダル)
    *   `--radius-btn`: `18px` (ボタン・入力フィールド・トグル・タブ)
    *   `--radius-pill`: `9999px` (バッジ・アバター・プログレスバー)
*   **余白 (Spacing)**: 8ptグリッドシステムを採用します。
    *   `8px` (極小・パーツ内余白), `16px` (小・コンテンツ間余白), `24px` (中・カード内パディング), `32px` (大・画面端マージン)
*   **影 (Elevation)**:
    *   ピンクの光彩を `alpha: 0.18` 〜 `0.28` で透過ブレンドし、浮遊感を表現します。

## 5. キャラクター ＆ アバターアセット定義
 
### 5.1. マスコット「ベリーちゃん」
*   **ビジュアル素材**:
    *   最小限の実装コストで愛着を持たせるため、いちご絵文字 🍓 および、CSS / SVG によるシンプルなベクターイラストを使用します。
*   **表情・アニメーション差分**:
    *   `standard` (😊/🍓): 通常表示、応援。
    *   `happy` (🎉/🌟): 正解コンボ達成時、セッション完了時。周囲にキラキラ (`Sparkle`) が舞う。
    *   `sad` (😢/💦): 「Again」の選択時、削除の「おわかれ確認」時。そっと寄り添う表情。

### 5.2. アカウントアバター画像アセット (共有デバイス・簡単ログイン用)
共有デバイスにおいて、子供を含むすべてのユーザーがキーボード入力なしで直感的にログインできるよう、アバター画像アセットを定義します。
*   **ビジュアルスタイル**: ぷっくりした丸みのある手描き調、または3Dクレイ（粘土）風の質感。背景はパステル系の優しいグラデーションの円形。
*   **初期アバターリスト**:
    1.  `avatar_strawberry.png` (いちご - ベリーちゃん)
    2.  `avatar_apple.png` (りんご)
    3.  `avatar_banana.png` (バナナ)
    4.  `avatar_bunny.png` (うさぎ)
    5.  `avatar_cat.png` (ねこ)
*   **ファイル形式と配置**: 解像度 128x128px 以上の透明アルファチャンネル付きPNGまたはSVG。プロジェクトの `public/assets/avatars/` ディレクトリに配置し、コンポーネントからスタティックに参照します。
*   **生成方法**: AIイメージジェネレータ（`generate_image`）を使用し、このビジュアルスタイルに沿って一貫したトーンでアセットを生成します。

## 6. 実装用 `:root` スニペット (index.css)

このスニペットは、そのまま [index.css](file:///src/client/index.css) に適用可能です。

```css
:root {
  /* Brand Colors */
  --berry-500: #FF6FA5;
  --berry-400: #FF8FB8;
  --berry-600: #F0568F;
  --berry-200: #FFC1D6;
  --berry-100: #FFE4EF;
  --berry-50:  #FFF0F5;

  /* Accent Colors */
  --mint-500:     #3FC7A0;
  --mint-100:     #DFF7EF;
  --lavender-500: #B69CFF;
  --sunny-400:    #FFC94D;
  --coral-500:    #FF7A6B;

  /* Neutral Colors */
  --ink-900:      #4A2C3A;
  --ink-700:      #6E5561;
  --ink-400:      #A38C97;
  --surface:      #FFFFFF;
  --surface-tint: #FFF7FA;
  --bg-gradient:  linear-gradient(180deg, #FFF0F5 0%, #FFE4EF 100%);

  /* Shape */
  --radius-card: 28px;
  --radius-btn:  18px;
  --radius-pill: 9999px;

  /* Elevation */
  --shadow-soft:  0 8px 24px rgba(255, 111, 165, 0.18);
  --shadow-pop:   0 12px 32px rgba(255, 111, 165, 0.28);
  --shadow-press: 0 2px 8px rgba(255, 111, 165, 0.20);

  /* Focus Indicator (Accessibility) */
  --focus-ring:   0 0 0 4px rgba(255, 111, 165, 0.4);
}

```
