## 2. Quick Wins（高インパクト×低コスト＝最優先で着手）

1. **F-5 文言のやさしい統一**（CSS不要、文字列置換中心）
2. **F-6 Again の非難しない配色/記号**（配色トークン適用で解消）
3. **F-7 進捗の0始まり是正＋励まし**（表示ロジックのみ）

→ これらは小コストで「冷たさ・罰的印象」を即改善できる。

## 3. 最大インパクトの主戦場

- **F-1 正解フィードバック**（感情の最大の谷を山に変える）＋ それを支える **F-11 useSound**
- **F-2/F-3 ピンク化＋トークン化**（第一印象と一貫性の土台）

この2軸が本リニューアルの核。

## 4. 設計フェーズ（/ux-design）への申し送り（決めるべき論点）

- **配色トークンの具体値**: ピンクの階調・正解カラー（緑→ミント）・状態色の確定 → `visual-design.md`（Berryパレット済み。実装値の妥当性を最終確認）
- **正解音の実装方式**: Web Audio合成（ファイル不要）か短い音源か → `sound-haptics.md`
- **iOS Safari 対応**: AudioContext解錠タイミング（初回タップ）・vibrate存在チェック
- **マスコット採否と実装コスト**: 絵文字🍓 or SVG、表情差分の範囲
- **ストリーク/継続の保持先**: localStorage で足りるか（サーバー拡張は範囲外）
- **Auto-Play UIの再配置方針**: 折りたたみ or 別セクション
- **Atomic Design 移行範囲**: どこまで既存を再配置するか（コスト見合い）
- **Could（XP/レベル/カレンダー）はV1で作らない**ことの再確認（スコープ厳守）

## 5. 結論

現状は「機能◯・体験✕」。改善は色替えに留めず、
**①正解の瞬間に山を作る（音・演出・言葉）② ピンク世界観＋トークン基盤 ③ 完了のクライマックス**
の3点を軸に、Quick Win（文言・Again配色・進捗）を先行して着手するのが最短で効く。
