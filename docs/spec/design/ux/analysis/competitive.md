# UI/UX分析 - STEP 4: 競合 / 参照UI 比較

本ドキュメントは、一般的な言語学習アプリ（Duolingoやmikan、Ankiなど）の体験設計（ゲーミフィケーション・モチベーション維持）と比較し、現状のアプリに取り入れるべき良質なパターンと、世界観を守るために避けるべきパターンを抽出したものです。

---

## 1. 競合・参照UIとの比較分析

| 観点 | 競合・参照UIの傾向（例: Duolingo, mikan） | 現状のアプリ | 差分からの学び / 改善方針 |
|:---|:---|:---|:---|
| **第一印象<br>（色/トーン）** | 親しみやすいフラットデザイン。白背景にブランドのビビッドな原色（黄・緑など）が多く、元気だがややシステム感が残る。 | Berryテーマ（ピンク基調、深みのあるブラウン `--ink-900`）による、温かみのあるパステル調の世界観。 | **【維持・強化】** 現状の世界観は競合と差別化できる最大の強み。より高級感と温かみを持たせるため、この色調と世界観を徹底的にベースとする。 |
| **リスト表現** | 既習/未習/苦手な単語が一目でわかる視覚的ラベル（星、炎、カラーインジケータ）があり、情報密度が低くスクロールしやすい。 | ページネーション付きのシンプルな縦リスト。ステータスはバッジ表示されているが、リスト全体の情報密度が高く文字が詰まり気味。 | **【取り込む】** 各単語カードの余白を広げ、ステータス（Review等）に応じて背景の透過度やボーダーカラーを変え、スクロール時に直感的に「弱点」を捉えられるようにする。 |
| **フィードバック<br>（正解演出）** | キャラクターがアニメーションで喜び、コンボ時には「Excellent!」等のポップアップや弾むエフェクトが即座に走る。 | `CelebrationOverlay` や `SuccessToast` が機能しているが、カード自体やボタンの物理的なアニメーション（弾み等）は控えめ。 | **【取り込む】** 正解時に回答ボタンやカード自体が「ぷるん」と弾むマイクロアニメーション（Spring物理挙動）をCSSで実装し、指先への快感を高める。 |
| **継続の仕掛け** | 連続学習日数を強力にプッシュ。途切れそうなときにキャラクターが泣きそうな顔でリマインドするなど、感情に訴える。 | 連続日数ストリークはあるが、完了画面やヘッダーに小さく数字が表示される程度で、学習を忘れた日の引き止め体験は弱い。 | **【取り込む（やさしい声かけ）】** ストリークが継続している時はベリーちゃんが大きく褒め、途切れた日には責めるのではなく「またいっしょにはじめよう🍓」と寄り添う体験を設計する。 |
| **情報密度・余白** | 1画面あたりの情報量が極めて少なく、文字サイズは大きく、タップターゲットは最低でも48px以上確保され、余白が広い。 | モバイル画面内に多くの例文やテキストが収まっている。一部のボタンやフォーム入力欄の間隔が狭く、誤タップが発生しやすい。 | **【取り込む】** 片手操作の親指領域を考慮し、ボタン間隔を1.5倍に広げ、文字サイズもモバイルに最適化して情報密度を下げる（美的ミニマリズム）。 |

## 2. 体験設計における「取り込むべきパターン」と「避けるべきパターン」

### 取り込むべきパターン (Do)
1.  **感覚的・身体的なフィードバック (Micro-interactions)**:
    *   ボタンをタップした際の「ぷにっ」とした凹みや、正解時の「シャキーン」とカードが消えるトランジションなど、触感的な気持ちよさ。
2.  **今日のごほうび（ノルマ達成の瞬間）**:
    *   「今日の目標（例: 10語学習）をクリアした！」という日々のマイルストーンを完了画面で大きく称え、達成感を習慣化する。

### 避けるべきパターン (Don't)
1.  **冷たく厳しいペナルティ演出（Anki等の無機質さ）**:
    *   「間違えたら最初からやり直し」といった厳しい制約や、暗いシステム警告ダイアログは、アプリの持つ「やさしい応援トーン」を台無しにするため避ける。
2.  **複雑な統計グラフの羅列**:
    *   学習進捗を数値のグラフやデータで過度に分析表示することは、対象ユーザー（癒やしと隙間時間の学習を求める層）に認知的疲労を与えるため、シンプルな進捗バーと励ましのメッセージに置き換える。
