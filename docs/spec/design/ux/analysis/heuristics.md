# UI/UX分析 - STEP 2: ヒューリスティック評価

本ドキュメントは、Nielsenの「UIデザインにおける10のヒューリスティックス」およびモバイル特有の操作観点に照らし、現状のUIの課題を評価したものです。（深刻度が高い順にソートしています）

---

## 1. 深刻度：高 (High)

### [原則3: ユーザーの制御と自由] 学習中の巻き戻し不可
*   **事実**: フラッシュカードの学習中（クイズ進行中）、一度「Good（覚えた）」または「Again（もう一回）」を押すと、次のカードへ強制的に遷移し、直前のカードに戻るボタンが存在しない。
*   **なぜ問題か**: ユーザーが操作ミス（誤タップ）をした場合、取り消して戻ることができず、「さっきの単語をもう一度確認したい」という自己学習のコントロール欲求を阻害し、フラッシュカードの学習効率を低下させる。
*   **改善仮説**: 学習画面に「1つ前の単語に戻る（Undo）」ボタンを小さく配置し、タップミスからの回復を容易にする。

---

## 2. 深刻度：中 (Medium)

### [モバイル観点] 親指の到達範囲外にある主要アクション
*   **事実**: 単語セット選択画面の「＋ セットをつくる」ボタンや、単語リスト画面の「＋ 単語を追加」ボタンが、画面右上やリスト上部の親指が届きにくいエリアに配置されている。
*   **なぜ問題か**: ターゲットユーザーである「隙間時間・片手操作のiPhoneユーザー」にとって、親指を大きく伸ばす必要があり、片手操作時の落下リスクや身体的負荷を高める。
*   **改善仮説**: モバイル表示時は、画面の右下にフローティングアクションボタン（FAB）として浮かせる、あるいは画面下部に吸着させる配置を検討する。

### [iOS Safari制約] 初回発音のAutoplayブロックによる無音現象
*   **事実**: ユーザー選択直後に学習画面をリストモードではなく学習モードで開始した場合、または自動再生を開始した場合、ユーザーが明示的に画面を一度もタップしていないと、iOS Safariの自動再生ポリシーによって発音音声（TTS）や Oscillator 音がブロックされ無音で進行してしまう。
*   **なぜ問題か**: 音声ありきの言語学習において、バグのように感じられ、体験の質（UX設計における体験加点（Plussing）の観点）が低下する。
*   **改善仮説**: セッション開始時に「スタート！」などの明示的な大きなボタンをユーザーにタップさせ、そのユーザーアクティベーションイベントを通じて確実にAudioContextとTTSをアンロックする。

## 3. 深刻度：低 (Low)

### [原則1: システム状態の可視化] 自動再生の残り時間不明
*   **事実**: 自動再生モード中、カードがめくられたり次の単語に進むまでの待機秒数（残り時間）が視覚的に示されない。
*   **なぜ問題か**: ユーザーは次にいつ発音やフリップが発生するか予測できず、自分のペースで画面を見て良いのか不安になり、自動再生のリズムにシンクロしにくい。
*   **改善仮説**: カードの周囲またはヘッダー部に、次の動作までのカウントダウンを示す「円形プログレス」や「ローディングバー」を表示する。

### [原則4: 一貫性と標準] フォームコントロールのネイティブ浮き
*   **事実**: セット作成フォームや単語作成フォームの `select` や `input` 要素が一部ブラウザデフォルト（ネイティブ）に近い見た目になっており、他のBerryテーマの可愛らしい角丸や影と調和していない。
*   **なぜ問題か**: アプリ全体の統一された世界観（Theming）において、突然システムライクなUIが現れるため没入感が薄れる。
*   **改善仮説**: フォームのセレクトボックス等をカスタムドロップダウンに置き換え、Berryテーマのデザイントークン（`--radius-btn` とピンクのフォーカスライン）を完全に適用する。

### [原則10: ヘルプとドキュメント] チュートリアルの不足
*   **事実**: 初めてアプリを開いて学習を開始する際、フラッシュカードの操作（めくってから判定を選ぶという流れ）についての説明やチュートリアルが一切ない。
*   **なぜ問題か**: 単語学習アプリに慣れていない初心者が、めくる前の裏面を見て回答するなどの誤った方法で使い始めてしまう可能性がある。
*   **改善仮説**: 初回学習開始時に、1スライドで完了するイラスト付きの「めくりかたガイド」をモーダルで表示する。
