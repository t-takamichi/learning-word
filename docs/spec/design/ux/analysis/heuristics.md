# 分析 STEP2 — ヒューリスティック評価

Nielsen 10原則＋モバイル観点で減点法診断。深刻度順（高→低）／親: [../README.md](../README.md)

各項目: `事実 → なぜ問題か → 深刻度 → 改善仮説`

---

## 深刻度：高

### H-01 [#1 状態の可視化] 正解しても何も返らない
- **事実**: Goodタップ時、`submitReview('good')` を呼ぶのみで、音・アニメ・言葉のフィードバックが皆無（`ReviewButtons.tsx`）。
- **なぜ問題**: 「できた」が体で分からず、学習の達成感＝継続動機が生まれない。無反応は"手応えのなさ"として離脱要因になる。
- **深刻度**: 高
- **改善仮説**: Goodに音(Nice!)＋弾みアニメ＋褒めトーストの3チャンネルを即時発火する。

### H-02 [#4 一貫性] 配色トーンが対象ユーザーに合っていない
- **事実**: 背景 `#0c0d12` の黒基調＋白文字＋グレー。全画面が無機質。
- **なぜ問題**: 対象（20代女性・隙間時間）に感情的に響かず、「開きたい」気持ちを作れない。ユーザー要望とも乖離。
- **深刻度**: 高
- **改善仮説**: ピンク基調のやさしいテーマへ。トークン化して全画面の一貫性を担保。

### H-03 [#2 実世界との一致] 文言が英語混在で冷たい
- **事実**: 「Auto-Play: ON」「Front/Back」「Good/Again」等の英語UIと VI/JA が混在（`StudyPage.tsx`）。
- **なぜ問題**: 学習者が"操作対象の英語"と"UIの英語"を混同しうる。命令的で温かみがない。
- **深刻度**: 高
- **改善仮説**: UI文言を統一（やさしい応援形）。「できた！」「もういちど」等。

### H-04 [#8 美的・ミニマル] Auto-Play制御が主役を食っている
- **事実**: 秒数の数値入力を含む自動再生UIがヘッダー上部に常時露出（`StudyPage.tsx:98-134`）。
- **なぜ問題**: 学習の主役（カード）より技術的設定が目立ち、"エンジニアのツール"印象を強める。情報過多。
- **深刻度**: 高
- **改善仮説**: Auto-Play設定を折りたたみ/別UIへ退避し、既定は隠す。

## 深刻度：中

### H-05 [#1 可視化] 進捗が0始まりで「0 / 10」表示
- **事実**: `currentIndex / words.length` を表示。開始直後は `0 / 10`（`StudyPage.tsx:140`）。
- **なぜ問題**: 「0」は達成感の初速を削ぐ。1枚目を「1/10」と感じたいユーザー心理に反する。
- **深刻度**: 中
- **改善仮説**: 「現在の枚数(index+1)」または「完了数」を意図に沿って表示。残り枚数の励ましも追加。

### H-06 [#4 一貫性] 完了画面が"締め"になっていない
- **事実**: 静的な🎉絵文字＋テキスト＋再開ボタンのみ。音・コンフェッティ・サマリー無し（`CompleteScreen.tsx`）。
- **なぜ問題**: セッションのクライマックスが弱く、"やりきった"余韻と再訪動機を作れない。
- **深刻度**: 中
- **改善仮説**: 祝福演出＋その日のサマリー＋マスコットの応援を追加。

### H-07 [#6 認識より想起 / #2] Again=❌・Good=⭕ が誤解を生む
- **事実**: Again に ❌（バツ）、Good に ⭕（マル）アイコン。
- **なぜ問題**: 自己評価（覚えていたか）なのに、○×＝正誤採点の記号は「間違い＝叱られる」印象。失敗を罰と感じさせる。
- **深刻度**: 中
- **改善仮説**: 記号と色をやさしく（Again=やわらかコーラル/「もういちど」、Good=ミント/「できた！」）。責めない配色。

### H-08 [モバイル] Goodボタンが白で押し心地が弱い
- **事実**: Goodは `#ffffff` 背景・`#1e1e24` 文字（`ReviewButtons.module.css:41-45`）。
- **なぜ問題**: 主要アクション（正解）が"白い箱"で高揚感がなく、押したくならない。
- **深刻度**: 中
- **改善仮説**: 主役CTAをブランド色の"ぷっくりボタン"にし、押下フィードバックを付ける。

## 深刻度：低

### H-09 [#8] 角丸・影の値がバラバラ
- **事実**: 角丸 8/12/14/16/24/28px が混在。影も黒系で複数値。
- **なぜ問題**: 統一感が弱く、丁寧さ（Craft）が伝わらない。
- **深刻度**: 低
- **改善仮説**: 角丸・影をトークン化して統一。

### H-10 [#7 効率性] 単語リストが学習画面に同居
- **事実**: `WordList` が StudyPage 下部に常時表示（`StudyPage.tsx:162`）。
- **なぜ問題**: 学習集中とスキャンの目的が同一スクロールに混在。役割が曖昧。
- **深刻度**: 低（設計判断。要望「10件表示」は満たしている）
- **改善仮説**: タブ/セクション分離は将来検討。まずは視覚的なまとまりを強化。

### H-11 [アクセシビリティ] reduced-motion 未対応
- **事実**: `@media (prefers-reduced-motion)` の記述なし。
- **なぜ問題**: 今後アニメを足す際、モーション過敏なユーザーへの配慮が抜ける。
- **深刻度**: 低（現状アニメが少ないため顕在化していない）
- **改善仮説**: 演出追加と同時に reduced-motion 簡略版を用意。

## 維持すべき良い点（壊さない）

- タップターゲット 44/48px を満たしている（ReviewButtons/AudioButton/pageButton）
- セーフエリア考慮が一部にある
- ステータスバッジ（new/weak/mastered）の色分けは機能的に有効 → 色をピンク系に寄せて活かす
- 進捗バー・言語トグルの構造は妥当 → 配色だけ差し替え
