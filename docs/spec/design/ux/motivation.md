# モチベーション・演出仕様 (Motivation & Delight)

本ドキュメントは、極上の「Delight（よろこび）」をもたらし、ユーザーがゲーム感覚で英語学習を習慣化できるよう、即時フィードバックと継続の仕掛け、およびやさしいマイクロコピーの設計を定義します。

---

## 1. 学習モチベーションループ (Motivation Loop)

学習者の習慣化は、以下の「トリガー」「アクション」「報酬」「投資」のサイクルで設計します。

```
[1. トリガー] (隙間時間 / ベリーちゃんのプッシュ)
     │
     ▼
[2. アクション] (カードをタップしてめくる、Good/Againを選択する)
     │
     ▼
[3. 報酬 (Delight)] (正解時のピコーン音、振動、キラキラのスパークル演出)
     │
     ▼
[4. 投資 (積み重ね)] (ストリーク日数の蓄積、単語セット完了の王冠👑)
```

---

## 2. 演出・機能の優先度 (Must / Should / Could)

過剰な演出による認知的疲労（**P4: やりすぎ防止**）を避けるため、実装の優先順位を定義し、Could 要素は今回の設計・実装対象外とします。

*   **Must (必須・即時実装)**:
    *   **正解（Good）への3チャネル・フィードバック**: 「音 (Correct音)」＋「振動 (Haptic)」＋「視覚 (Sparkleエフェクト)」を200ms以内の遅延で同期発動する。
    *   **失敗（Again）の低刺激ケア**: 不快なブザー等を完全に排除し、やさしい低音 Oscillator 音 (`again`) で寄り添う。
    *   **進捗の可視化**: セッション中の「すすんだ割合」を％とインジケータで常時表示。
*   **Should (推奨・V1実装)**:
    *   **連続正解コンボ (🔥)**: 5連続正解などの節目で `CelebrationOverlay` が作動。画面にキラキラとベリーちゃんのアニメーションを表示。
    *   **セッション完了の祝福**: アルペジオファンファーレ音とともに、王冠をかぶったベリーちゃんのお祝いを表示。
    *   **継続日数ストリーク**: 連続学習日数を記録し、ログイン時に炎マーク 🔥 で表示。
*   **Could (スコープ外・V1では作らない)**:
    *   **学習XP（経験値）やレベルシステム**: レベルアップ演出は複雑さを増すため今回は見送る。
    *   **カレンダー形式の学習実績スタンプ**: ユーザーセレクト画面や完了画面のシンプルさを保つため、見送る。

---

## 3. 正解演出（Delight）と失敗時の配慮

### 3.1. 正解（Good）フィードバック仕様
*   **反応速度**: タップ判定から **200ms以内** にすべてのフィードバックを開始する。
*   **音の演出**: 軽快な高音（C6→E6）の合成音 `correct`。
*   **振動の演出**: スマホが「チッ」と1回だけ心地よく揺れる瞬間的な微細ハプティクス（15ms）。
*   **視覚の演出**: タップされたカードの周囲から、半透明のピンクやイエローの星型・丸型パーティクル (`Sparkle`) が放射状に10本程度飛び出し、0.4秒でフェードアウトする。

### 3.2. 不正解・もう一回（Again）のやさしいケア
*   **デザイン意図**: ユーザーのプライドを傷つけず、「次にまた覚えれば大丈夫」という心理的安全性（**P5: 細部にやさしさ**）を確保します。
*   **音の演出**: 低音で落ち着いたサイン波のスライド音 `again`。
*   **視覚の演出**: コーラルピンク (`--coral-500`) をキーカラーにしつつ、点滅や強い警告アニメーションは一切行わない。

## 4. マイクロコピー対応表（応援形）

冷たい機械的メッセージや命令形を排除し、ベリーちゃんが語りかける「応援形・寄り添いトーン」に統一します。

| シチュエーション | 従来の表現（避けるべき❌） | Berryテーマの表現（推奨✅） | 意図 / 体験上の工夫 |
|:---|:---|:---|:---|
| **単語の削除確認** | 単語を削除しますか？<br>(元に戻せません) | この単語と、おわかれする？🍓<br>（いままでの学習データも消えちゃいます） | 「おわかれ」という情緒的表現により、データ削除の重みをやさしく伝達。 |
| **単語セットの削除確認** | 単語セットを削除しますか？ | この単語セットと、おわかれする？<br>（セットの中の単語もバイバイしちゃうよ💦） | セット削除に伴う子単語の消失を、感情的に配慮しながら警告。 |
| **単語登録の上限到達** | 登録上限に達しました。<br>これ以上登録できません。 | いまはここまで！よくがんばったね。<br>新しく追加するまえに、いまの単語をたくさん復習してあげよう🌸 | 登録拒否を「お祝い・ねぎらい」に変換し、復習（次の正しい行動）へ誘導。 |
| **単語セット登録の上限** | 単語セットは50個までです。 | いまはここまで！よくがんばったね。<br>新しく追加するまえに、いまの単語セットをたくさん復習してあげよう🌸 | 同様に、セット作成上限時にも優しく労い、既存セットの活用を促す。 |
| **未学習の空状態** | 登録されている単語はありません。 | まだ単語が登録されていません。<br>おぼえたい単語を、ここから追加してね🍓 | 「これから一緒に始めよう」という前向きな呼びかけ。 |

## 5. アニメーションとアクセシビリティの配慮 (`prefers-reduced-motion`)

*   **アニメーション時間制限**:
    *   コンボ祝福やセッション完了時のアニメーションは、ユーザーの操作を長時間拘束しないよう、最大でも **2.5秒以内** に完全に収束（フェードアウト）し、自動的に次の画面（または操作可能状態）へ移行します。
*   **`prefers-reduced-motion` の尊重**:
    *   OSの「視覚効果を減らす」設定が有効になっている場合は、カードフリップの回転速度を2倍にし、コンボ・セッション完了時のパーティクル飛散などの激しいアニメーションはすべてシンプルなフェードイン/アウトに置き換えます。
