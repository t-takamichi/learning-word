# Motivation & Gamification — モチベーション機構

親: [README.md](./README.md)

「正解したら、モチベーションが上がらないと」という要望への回答。
学習の各瞬間に **"報われた"** という感情を設計で作り込む。

---

## 1. モチベーション設計の全体像

継続を生むループ（Hook Model 的に）:

```
きっかけ ──▶ 行動 ──▶ ごほうび ──▶ 積み重ね ──▶（また戻る）
（開く）    （1枚こたえる） （音/演出/褒め）（ストリーク/XP） （明日の自分が得をする）
```

各機能を優先度付きで定義する。

| 機構 | 優先度 | 一言 |
|------|--------|------|
| 正解フィードバック（音・アニメ・言葉） | **Must** | 正解の瞬間に必ず"うれしい" |
| セッション進捗の可視化 | **Must** | あと何枚かが見える安心 |
| 連続正解コンボ（ストリーク） | **Should** | 🔥 途切れさせたくない気持ち |
| 完了時の祝福（コンフェッティ＋マスコット） | **Should** | やりきった達成感 |
| デイリーゴール & 継続日数 | **Should** | 「今日もできた」の記録 |
| XP / レベル / スタンプ | **Could** | 長期の成長実感 |
| 週間カレンダー（学習の足あと） | **Could** | 積み重ねの可視化 |

---

## 2. 正解フィードバック（Must）

**トリガー**: 裏面で「できた！（Good）」をタップした瞬間。

3チャンネル同時発火（マルチモーダル）:

| チャンネル | 内容 |
|-----------|------|
| 🔊 音 | 明るい上昇系の "**Nice!**" 音（[sound-haptics.md](./sound-haptics.md)） |
| ✨ 視覚 | カードが軽く弾む（scale 1.0→1.04→1.0, 220ms）＋ミント色のチェック＋粒（スパークル数個）が舞う |
| 📳 触覚 | 軽い成功ハプティクス（対応端末のみ。iOS Safari制約は sound-haptics 参照） |
| 💬 言葉 | 短い応援トースト（下から出て自動で消える）：ランダムに `Nice! / いいね！ / その調子！ / さすが！` |

**Again（もういちど）のとき**: 責めない。無音〜やわらかい低めのトン。文言は `だいじょうぶ、もういちど♪`。
→ 失敗を"罰"にしないことが継続の肝。
→ **現状の ❌（バツ）記号は廃止**（正誤採点＝叱られる印象になるため）。記号はハート/矢印など中立的なものにし、色は `--coral-500`（やわらかい）に。Good側の ⭕ も廃し、チェック＋「できた！」に（分析 F-6）。

---

## 3. セッション進捗（Must）

- 上部に進捗バー（[visual-design.md](./visual-design.md) 5.3）＋ `4 / 10` ピルチップ
- **表示は「完了した枚数」基準に是正**する（分析 F-7）。現状は0始まりの `currentIndex / 10` で開始時「0/10」と出て初速の達成感を削ぐため、1枚回答＝「1/10」と感じられる基準に統一する
- 残り2枚で「あと少し！」のマイクロコピーがバー横に出る（ゴールが近い高揚感）

---

## 4. 連続正解コンボ / ストリーク（Should）

- Good を連続で出すたびにカウント。`2連続` から `🔥2` バッジがヘッダーに出現し数字が伸びる
- しきい値で強めの祝福：**3 / 5 / 10連続** で `--sunny-400` のバースト＋専用の上昇音
- Again でコンボはリセット。ただし **リセットを大げさに描かない**（静かに0へ）。落胆させない

---

## 5. 完了時の祝福（Should）

**トリガー**: セッションの最後のカードを回答（`CompleteScreen` 表示）。

| 要素 | 内容 |
|------|------|
| 🎉 コンフェッティ | ピンク/ミント/サニーの紙吹雪が上から（2.5秒で自然に収束） |
| 🍓 マスコット | Berry が `cheer` 表情で登場し「ぜんぶできたね！」 |
| 🎵 ファンファーレ | 短い達成音（1.2秒以内） |
| 📊 サマリー | 「今日は N 枚／正解 M 枚／🔥最高コンボ K」を優しいカードで表示 |
| 🔁 次アクション | 「もういちど」「単語リストを見る」を大きめボタンで |

- `prefers-reduced-motion` 時: コンフェッティは静的なイラスト表示に置換。

---

## 6. デイリーゴール & 継続日数（Should）

- デイリーゴール（既定：1セッション＝10枚）。達成でヘッダーに ✅
- 継続日数（ストリークデイズ）：連続で学習した日数を `🔥N日` で表示
- **「途切れ回避」より「戻ってこられる優しさ」** を優先。途切れても責めるコピーは出さない

> **実装メモ**: 継続日数・デイリー達成は `localStorage` で軽量に持てる（サーバー拡張は任意）。
> 既存DBの review ログがあれば集計も可能だが、V1は端末ローカルで十分。

---

## 7. XP / レベル / スタンプ（Could）

- Good=+10XP / コンボボーナス。一定XPでレベルアップ（マスコットが少し育つ演出）
- スタンプ帳：「7日継続」「100枚達成」などのバッジ
- **V1では実装しなくてよい**。設計上の拡張余地として定義（過剰実装を避ける — [quality-bar.md](./quality-bar.md) P4）

---

## 8. マイクロコピー原則

「命令」ではなく「応援」。すべての文言を"やさしい日本語＋前向き"に。

| 場面 | ❌ 旧（硬い） | ✅ 新（やさしい） |
|------|--------------|------------------|
| 答えを表示 | Show Answer | こたえを見る |
| 正解 | Good | できた！ |
| 不正解 | Again | もういちど |
| 空状態 | No words | まだ単語がないみたい。追加してみよう🍓 |
| 完了 | Session Complete | ぜんぶできたね！おつかれさま🎉 |
| ローディング | Loading... / Đang tải | じゅんびちゅう… |
| コンボ | Streak x3 | 3れんぞく！すごい🔥 |
| 自動再生 | Auto-Play: ON/OFF | じどうめくり ON/OFF |
| 秒数設定 | Front / Back | おもて / うら（秒） |
| 進捗ラベル | Tiến độ (進捗) | いまここ |
| ユーザー選択 | Select User / Login | だれがまなぶ？🍓 |
| ユーザー追加 | Create User | あたらしくいっしょにまなぶ人 |
| ユーザー削除 | Delete User | おわかれする（削除） |
| レベル選択 | Select Level | どのレベルに挑戦する？ |
| レベル分類 | Basic / Intermediate / Advanced | はじめて（初級） / なれてきた（中級） / ばっちり（上級） |
| セット進捗 | Progress: 12/50 | すすんだ数：12 / 50語 |

> **英語混在の解消（F-5）**: 現状 `StudyPage.tsx` は「Auto-Play: ON」「Front/Back」等の英語UIが VI/JA と混在している。UI文言は**やさしい日本語ベースに統一**する（学習対象の英単語とUIの英語が混同されないため）。
> **新規画面の表現**: ユーザー選択画面、レベル（単語セット）選択画面の文言も、上記の「やさしい応援形」のルールを徹底する。

---

## 9. 受け入れ基準（この機構のAC）

- [ ] Good タップ時、音・視覚・言葉の3チャンネルが 200ms 以内に発火する
- [ ] Again はネガティブに演出されない（責めるUI・強い赤・エラー音を出さない）
- [ ] 進捗が常に「現在/全体」で分かり、残り2枚で励ましが出る
- [ ] 完了画面でその日のサマリーとマスコットの祝福が表示される
- [ ] すべての演出が `prefers-reduced-motion` で簡略化される
- [ ] 音は既定ONだが、ワンタップでミュートでき設定が保持される
