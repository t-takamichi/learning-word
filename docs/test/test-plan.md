# 詳細テスト計画書 ＆ 検証ケースマトリクス (Test Plan & Matrix)

本計画書は、アプリ内の各モジュール・機能が要件定義およびUI/UX設計に従って正常に動作しているかを判定するための、具体的な**テストケースと検証手順**をまとめた仕様書です。

---

## 1. 機能別テストマトリクス概要

| モジュールID | テスト対象領域 | 主な検証内容 |
|:---|:---|:---|
| **TC-01** | ユーザー認証・セキュリティ | 新規登録、ログイン、英数字パスワード対応、アカウント削除、ユーザー名非露出 |
| **TC-02** | 学習セッション ＆ カード操作 | カード反転、Good/Again判定、Undo（巻き戻し）、左右フリックスワイプ |
| **TC-03** | サウンド・ハプティクス ＆ 解錠 | iOS解錠「スタート！」ボタン、Web Audio API合成音、マナーモード動作 |
| **TC-04** | リザルト ＆ 継続ループ | 目標達成度ゲージ、おすすめセット一発遷移、交代ショートカット |
| **TC-05** | 単語・セット管理 | リスト表示、言語トグル（日/ベト）、バリデーションエラー |
| **TC-06** | モバイル制約 ＆ UX品質 | 16pxズーム防止、44pxタップ領域、--focus-ring、非ブロッキング演出 |

## 2. 詳細テストケース仕様

### TC-01: ユーザー認証 ＆ セキュリティ検証

| ケースID | テスト項目 | 操作手順 | 期待される結果 (Pass条件) |
|:---|:---|:---|:---|
| **TC-01-01** | 新規ユーザー登録 | 1. ログイン画面で「あたらしくはじめる」を選択<br>2. おなまえと英数字パスワードを入力して実行 | 新規アカウントが作成され、レベル選択画面へ遷移すること。 |
| **TC-01-02** | 英数字パスワードでのログイン | 1. 既存アカウントのおなまえと英数字パスワードを入力<br>2. 「ログインする」をタップ | 正常に認証され、レベル選択画面へ遷移すること。 |
| **TC-01-03** | パスワード誤り時の制御 | 誤ったパスワードを入力してログインを実行 | 「ユーザー名または合言葉(PIN)が正しくありません」とやさしく表示され、画面遷移しないこと。 |
| **TC-01-04** | **ユーザー非露出検証 (セキュリティ)** | 未認証の状態で `/users` 画面にアクセス | **他人のユーザー名やアバター一覧が画面上に一切表示されないこと**（未認証列挙の防止）。 |
| **TC-01-05** | アカウント削除 (おわかれ) | 1. ログイン後、「アカウントを削除する」をタップ<br>2. 確認ダイアログでパスワードを入力 | データが安全に削除され、ログアウト状態でログイン画面に戻ること。 |

---

### TC-02: 学習セッション ＆ カード操作検証

| ケースID | テスト項目 | 操作手順 | 期待される結果 (Pass条件) |
|:---|:---|:---|:---|
| **TC-02-01** | カード表面/裏面切り替え | カードまたは「こたえを見る」ボタンをタップ | カードがフリップアニメーションし、単語の意味と例文が表示されること。 |
| **TC-02-02** | Good 判定の動作 | カード裏面表示時に「できた！」をタップ | 1. `correct` 合成音が再生されること<br>2. 進捗インジケータとコンボ数が加算されること<br>3. 次のカードに進むこと。 |
| **TC-02-03** | Again 判定の動作 | カード裏面表示時に「もう一度」をタップ | 1. 低刺激な `again` 合成音が再生されること<br>2. コンボ数が 0 にリセットされること<br>3. 次のカードに進むこと。 |
| **TC-02-04** | **Undo (学習中巻き戻し)** | 判定直後に左上の Undo ボタンをタップ | 1. `undo` 音が鳴り、カードが1つ前の状態（裏面）に戻ること<br>2. 進捗数・正答数が安全に減算・復旧すること。 |
| **TC-02-05** | **フリックスワイプ操作** | カード裏面表示時に右または左へ大きくスワイプ | 1. 右スワイプで Good、左スワイプで Again が発動すること<br>2. 指の移動に合わせてカードが自然に追従傾斜すること。 |

---

### TC-03: サウンド・ハプティクス ＆ 音声解錠検証

| ケースID | テスト項目 | 操作手順 | 期待される結果 (Pass条件) |
|:---|:---|:---|:---|
| **TC-03-01** | **iOS Safari 音声解錠** | 画面起動後、学習セッションへ移動 | 「学習をスタート！🍓」ボタンが提示され、タップで AudioContext が活性化して無音化が回避されること。 |
| **TC-03-02** | Web Audio API 合成音 | Good / Again / Undo / Complete 実行 | 外部音声ファイルの読み込み遅延なく、200ms以内でピッチの異なる合成音が再生されること。 |
| **TC-03-03** | ミュート機能の保持 | サウンドトグル（🔊 / 🔇）をタップ | セッション音が消音され、リロード後も `localStorage` 経由で消音状態が維持されること。 |
| **TC-03-04** | バイブレーションの安全性 | Vibration API 非対応端末（iOS等）で操作 | JSエラーでクラッシュすることなく、安全にスルーされること。 |

### TC-04: リザルト ＆ モチベーション継続ループ検証

| ケースID | テスト項目 | 操作手順 | 期待される結果 (Pass条件) |
|:---|:---|:---|:---|
| **TC-04-01** | **目標達成度ゲージ** | セッション最後のカードを完了させる | リザルト画面で「きょうの目標達成度 🌟」がグラデーションアニメーション付きで表示されること。 |
| **TC-04-02** | おすすめセットダイレクト遷移 | 「次のおすすめセットに挑戦！🌸」をタップ | 一発で次のレベルの学習セッションが初期化・ロードされること。 |
| **TC-04-03** | 交代ショートカット | 「👤 つぎはだれの番？」をタップ | 1. `tap` 音が鳴り安全にログアウトされること<br>2. 即座にログイン画面へ遷移すること。 |

### TC-05 ＆ TC-06: モバイルUI/UX・アクセシビリティ検証

| ケースID | テスト項目 | 操作手順 | 期待される結果 (Pass条件) |
|:---|:---|:---|:---|
| **TC-06-01** | **iOS フォームズーム防止** | スマホブラウザで入力欄をタップ | 入力要素の `font-size` が 16px 以上で保持され、画面の勝手なズームが起きないこと。 |
| **TC-06-02** | **フォーカスインジケータ** | Tabキー等でボタンや入力欄にフォーカス | ブラウザ標準の青枠が排除され、Berryテーマの `--focus-ring`（ピンク光彩）が表示されること。 |
| **TC-06-03** | **非ブロッキング Celebration** | 5コンボ達成時の演出を確認 | 画面右下に非同期でポップアップし、演出中も背後のカード操作が一切ロックされないこと。 |
| **TC-06-04** | やさしいマイクロコピー | アカウント削除・単語削除ダイアログを開く | 命令形（「削除しますか？」）ではなく「おわかれする？」等のやさしいコピーが表示されること。 |

## 3. テスト実行の手順

1. **静的検証**:
   ```bash
   npm run typecheck   # 型エラー 0 件の確認
   npm run build       # 生産ビルドの通過確認
   ```
2. **ローカルサーバー起動**:
   ```bash
   npm run dev         # 開発サーバーの起動
   ```
3. **ブラウザおよびモバイル実機でのケース消化**:
   - Chrome デベロッパーツール（iPhone SE / iPad 描画モード）でのレイアウト・タップ領域検証。
   - 実機（iOS Safari / Android Chrome）での音声解錠・バイブレーション・スワイプ検証。
