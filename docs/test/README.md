# 全体テスト戦略書 (Test Strategy)

本ドキュメントは、単語学習アプリケーション「learning-word」における機能の正常性、セキュリティ、パフォーマンス、およびモバイル特有のUX品質を保証するための**テスト戦略と品質推進方針**を定義します。

---

## 1. テストの目的と品質目標

1.  **基本機能の決定論的保証**:
    ユーザー認証、単語セットの選択、学習セッション、レビュー記録などのコアロジックが正常に動作することを保証する。
2.  **セキュリティとプライバシーの厳格な保護**:
    未認証状態でのユーザー列挙脆弱性（ユーザー名一覧の露出）を完全に排除し、パスワード/PIN認証の安全性を維持する。
3.  **モバイル・デバイス特有制約の完全突破**:
    iOS Safari 等における AudioContext 自動再生制約、マナーモード動作、フリック/スワイプ操作、フォームフォーカス時の画面ズーム防止などを確実に検証する。
4.  **Berryテーマ UI/UX 演出品質の堅持**:
    遅延のないフィードバック（200ms以内の音・振動同期）、非ブロッキングなお祝い演出、やさしい応援形マイクロコピーの一貫性を保つ。

---

## 2. テストピラミッドと検証範囲

```
       / \
      / E2E \       <- Playwright / 手動実機検証 (ユーザーシナリオ・モバイル操作)
     /-------\
    /  統合   \      <- Hono APIルートテスト / Component Integration (React Query)
   /-----------\
  /   単体     \     <- Vitest (ドメインロジック / Web Audio SFX / セッション状態)
 /---------------\
```

### 2.1. レイヤー別テスト区分

| テストレベル | 目的・対象 | 検証手法 | 担当 |
|:---|:---|:---|:---|
| **単体テスト (Unit)** | ドメインロジック（`sfx.ts`, `useSession`, 各種ユーティリティ）の検証 | Vitest / Jest | 開発者 |
| **統合テスト (Integration)** | Hono API エンドポイント、SQLite DB トランザクション、React Query キャッシュ同期 | Vitest + Supertest | 開発者 |
| **E2E / 手動UXテスト** | ブラウザ・実機上の完全なユーザーシナリオ、タッチ/スワイプ、音声アンロック | Playwright / 手動実機チェック | 開発者 / QA |

---

## 3. 品質ゲート (Quality Gates)

コードの変更およびリリース時には、以下の条件をすべてクリアすることを必須とします。

- [ ] **型チェック (Type Check)**: `npm run typecheck` (`tsc --noEmit`) でエラーが 0 件であること。
- [ ] **ビルド検証 (Production Build)**: `npm run build` (`vite build`) が警告なしで正常完了すること。
- [ ] **セキュリティ検証**: 未認証でのユーザー名一覧返却 API（`GET /api/users` 等）が存在しないこと。
- [ ] **アクセシビリティ・デザイン検証**:
  - 生のHEX色コード指定（`#000000` 等）が存在せず、デザイントークンが適用されていること。
  - すべてのインタラクティブ要素のタップターゲットが 44px × 44px 以上確保されていること。
  - 入力要素の `font-size` が 16px 以上に設定され、iOS Safari でズームが発生しないこと。

---

## 4. 自動化推進ロードマップ

現状の手動検証・型チェック体制から段階的に自動テストを拡張します。

*   **フェーズ 1 (現行)**:
    *   `npm run typecheck` および `npm run build` による静的検証の徹底。
    *   [test-plan.md](./test-plan.md) に基づく体系的な手動・実機テストケースの運用。
*   **フェーズ 2 (短期)**:
    *   `Vitest` を導入し、`useSession` の巻き戻し（Undo）や `sfx.ts` の状態決定ロジックの単体テスト自動化。
*   **フェーズ 3 (中長期)**:
    *   `Playwright` を導入し、ログイン → レベル選択 → カード学習 → リザルトの主要E2Eフロー自動化。
