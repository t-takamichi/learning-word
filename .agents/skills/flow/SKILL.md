# SKILL: Flow（E2E自律型開発オーケストレーター）

## 概要

「〇〇の機能を変更したい」「このUI/UXを追加して」といったユーザーの自然言語による指示、あるいはコード変更を起点として、以下のプロセスを一切の手動介入なしに全自動で連続実行するスキルです。

```
[ユーザー指示] 
   └── 1. 設計書の自動更新 (design-sync / ux-design)
         └── 2. 影響分析・フェーズ再定義 (impl-phase / ux-impl-plan)
               └── 3. 実装計画書 (plan.md) の自動修正 (impl-plan / ux-impl-plan)
                     └── 4. コード自動実装とビルド・テスト検証 (impl / ux-impl)
```

## 起動方法

```bash
/flow [変更内容の指示]
/flow --reverse           # コードの直近差分から設計書へ逆同期する
```

* 例: `/flow "フラッシュカード画面に日本語翻訳の表示/非表示を切り替えるトグルボタンを追加して"`

---

## 前提条件

このスキルを実行する前に、以下が満たされている必要があります。

1. **Gitのクリーン状態**: 予期せぬ破壊やロールバックを安全に行うため、未コミットの変更がないこと（無い場合は事前にスタッシュを促します）。
2. **既存設計ドキュメントの存在**: `docs/spec/design/README.md` または `docs/spec/design/ux/` が存在していること。

## 実行ステップ (Skills)

以下のステップを**この順番で**連続して自律実行します。途中で検証エラーが発生した場合は、自己修復ルールに従って処理します。

| STEP | ファイル | 内容 | 出力・遷移先 |
|------|---------|------|------------|
| **1** | `skills/step1-design-sync.md` | ユーザーの指示から設計書の該当部分を特定し自動更新する | `docs/spec/design/` または `docs/spec/design/ux/` の更新 |
| **2** | `skills/step2-phase-sync.md` | 設計書の差分から影響フェーズを特定し、フェーズ定義を更新する | `docs/spec/impl/phase/` の更新 |
| **3** | `skills/step3-plan-sync.md` | 影響を受けるフェーズの実装計画書（`plan.md`）にタスクを自動マージする | `docs/spec/impl/steps/<番号>/plan.md` の更新 |
| **4** | `skills/step4-impl-sync.md` | 実装計画書に従ってコードを自動実装し、ビルド（tsc）・テストを実行する | `src/client/`, `src/server/` 等の修正・検証 |

---

## 共通ルール

E2E自動開発の進行にあたっては、以下の安全・修復ルールを厳格に守ってください。

```
.agents/skills/flow/references/rules/flow-rules.md
```

---

## 完了条件

STEP 4 が完了し、ビルドおよび自動テストがすべてグリーン（合格）になった時点で「E2E自律開発完了」を宣言し、最終的な変更内容（設計・計画・コードの差分）をユーザーに要約して報告します。

検証が3回以上の修復でも失敗した場合は、自動的に変更したコードを Git でロールバックし、エラーの詳細を報告して人間に処理を引き継ぎます。
