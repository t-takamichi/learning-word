# STEP 3: 実装計画書（plan.md）の自動同期 (plan-sync)

**目的**: 影響を受けるフェーズの実装計画書（`plan.md`）に、要件を満たすための新規タスクを自動マージ・更新する。

## 実施内容

1. **計画書の読み込み**:
   - 前ステップで特定された、影響を受けるフェーズ番号 N に対応する実装計画書（`docs/spec/impl/steps/N/plan.md`）を読み込む。

2. **追加タスクの定義**:
   - 設計変更およびフェーズ定義の差分から、新要件を満たすために必要な「実装コード変更」「新規コンポーネント」「APIエンドポイント追加」などのタスクを整理する。

3. **タスクの自動マージ**:
   - 既存のタスク（T1, T2...）のIDや順序を破壊しない。
   - 新規タスクを挿入・追記する際は、IDに `T-ADD-1`, `T-ADD-2` のような形式を使用し、タスク一覧テーブルの依存関係に基づいた適切な位置に追加する。
   - 各追加タスクに「成果物（ファイル）」「内容」「層」「検証方法」を明記する。
   - 必要に応じて、計画書の「前提・依存」や「完了条件（Doneの定義）」にも、新規の受け入れ基準を追記する。
   - ファイルを保存する。

4. **結果の報告**:
   - ユーザーに対し、「実装計画書 `docs/spec/impl/steps/N/plan.md` に新規タスク [タスクID] をマージしました」と報告する。

## 完了条件
- 影響を受ける `plan.md` が、既存の計画構成を崩すことなく、新規タスクを正しくマージして保存されていること。
- マークダウンのフォーマットが崩れていないこと。
