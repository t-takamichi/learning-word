# STEP 4: 自動実装・検証・自己修復・ロールバック (impl-sync)

**目的**: 更新された実装計画書に従ってコードを自律実装し、ビルドやテストで検証する。エラー時は自己修復を行い、困難な場合は安全にロールバックする。

## 実施内容

1. **実装対象の特定とバックアップ**:
   - `plan.md` に追加された新規タスク（例: `T-ADD-1`）の「成果物（ファイル）」を確認する。
   - コードを変更する前に、必ず `scripts/rollback.ts` の機能を使って対象ファイルのバックアップを作成する。
   - 例: `npx tsx .agents/skills/flow/scripts/rollback.ts backup [変更対象のファイルパス]`

2. **自律実装の実行**:
   - `plan.md` のタスク内容、および適用すべき設計、規約（`docs/rules/`配下）に従って、ソースコードを自律的に修正する。

3. **コンパイル・テスト検証**:
   - 実装完了後、`scripts/runner.ts` を用いて型チェックやテストを実行し、その成否を確認する。
   - 例: `npx tsx .agents/skills/flow/scripts/runner.ts "yarn typecheck"`

4. **自己修復（セルフリペア）ループ**:
   - もし検証エラー（型エラーやテスト失敗）が検出された場合、以下のループを最大 **3回** まで実行する。
     1. エラー内容（`runner.ts` の stderr や stdout のログ）を読み込み、原因を特定する。
     2. 原因に基づいてコードを再修正する。
     3. 再度 `runner.ts` で検証を実行する。
   - 3回繰り返してもエラーが解消しない場合は、自律修復を断念する。

5. **ロールバックまたはクリーンアップ**:
   - **検証合格（パス）時の処理**:
     - 作成した一時的なバックアップファイルをクリーンアップ（削除）する。
     - 実装成功をメモリに記録し、完了報告に進む。
   - **検証不合格（3回失敗）時の処理**:
     - 直ちに `scripts/rollback.ts` を用いて、変更を加えた対象ファイルをすべて元の状態にロールバックする。
     - 例: `npx tsx .agents/skills/flow/scripts/rollback.ts rollback [変更対象のファイルパス]`
     - 失敗したエラー内容とロールバック完了をログに明示し、ユーザーに引き継ぐ。

## 完了条件
- コードの修正が完了し、`yarn typecheck` やテストがすべて合格（グリーン）していること。
- または、検証失敗時に元のクリーンな状態に完全に復元（ロールバック）されていること。
