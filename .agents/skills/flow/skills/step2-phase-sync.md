# STEP 2: 影響分析・フェーズ定義更新 (phase-sync)

**目的**: 設計書の差分（Markdown）を分析し、影響を受ける実装フェーズ定義を特定・更新する。

## 実施内容

1. **影響フェーズの特定**:
   - 前ステップで更新された設計ファイル名をもとに、`scripts/impact-analyzer.ts` を実行して影響のあるフェーズ番号、およびフェーズ定義ファイル（`docs/spec/impl/phase/phase-N.md`）を特定する。
   - 例: `npx tsx .agents/skills/flow/scripts/impact-analyzer.ts [更新された設計ファイルのパス]`

2. **フェーズ定義の読み込み**:
   - 特定されたフェーズ定義ファイル（例: `docs/spec/impl/phase/phase-3.md`）を読み込む。

3. **フェーズ定義の更新**:
   - 設計書に追加された要件（例: 「単語のリストを10件表示」など）を、特定されたフェーズの「ゴール」および「スコープ（実装項目）」セクションに追記する。
   - 変更の際は、既存のゴールやスコープを破壊せず、追記・マージの形で行う。
   - 更新内容をファイルに保存する。

4. **結果の報告**:
   - ユーザーに対し、「フェーズ定義 `docs/spec/impl/phase/phase-N.md` を更新しました」と報告する。
   - 更新したフェーズ番号をメモリ（または次のステップへのインプット）として保持する。

## 完了条件
- 影響を受ける `phase-N.md` が、設計変更の内容を網羅した形で正しく更新・保存されていること。
- マークダウン構造が崩れていないこと。
