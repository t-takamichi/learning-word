# STEP 1: 計画書の読み込み・実装ルール確認

**目的**: 実装を始める前に、対象フェーズの計画とルールを完全に把握する。

## 実施内容

### 対象計画書の特定
- 引数のフェーズ番号 `N` を確認する
- `docs/spec/impl/steps/N/plan.md` を読み込む
  - 参照 / ゴール / 前提・依存 / タスク一覧 / 各タスク詳細 / Done定義

### 前提・依存の確認
- 計画書「前提・依存」に記載された前提フェーズが実装済みかを確認する
- 未実装の前提があれば、先にそれを処理するよう案内する
- 必要な依存パッケージ・環境が整っているかを確認する

### 実装ルールの読み込み（必須）
以下を実装前に必ず読み込み、違反しないことを確認する。

```
docs/rules/architecture.md   ← レイヤー・依存方向・DIパターン
docs/rules/TypeScript.md     ← 型・コーディング規約
docs/rules/hono.md           ← API実装規約
docs/rules/react.md          ← フロントエンド規約
```

### タスク順序の確定
- 計画書のタスク一覧（T1, T2, ...）を、実装順に並んでいることを確認する

## 出力
このSTEPはファイル出力なし。
タスク一覧と適用ルールを次STEPの実装へ引き継ぐ。
