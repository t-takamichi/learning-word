# Quality Gate — UX分析 完了チェックリスト (マルチエージェント版)

全項目を満たしたら「分析完了」とし、次の設計フェーズへ進む。

## 成果物
- [ ] `analysis/inventory.md`（現状トークンの実値・CSS定義付き）
- [ ] `analysis/heuristics.md`（深刻度 ＋ **PMによる具体的なファイルパス・行数特定付き**）
- [ ] `analysis/emotion.md`（**Mermaid グラフによる感情曲線** ＋ ゲーミフィケーション診断）
- [ ] `analysis/competitive.md`（参照UIとの比較 ＋ **デザイナーによる加点提案付き**）
- [ ] `analysis/findings.md`（優先度付き課題 ＋ 改善仮説 ＋ 設計への申し送り）

## 内容・品質チェック
- [ ] **ユーザーヒアリングの反映**: STEP 0 でユーザー（オーナー）と合意したターゲット属性やビジネス目標が前提として明記され、分析全体に貫かれているか。
- [ ] **事実とコードの紐付け**: すべての課題において、印象論ではなく具体的なファイル名、行数、またはCSS定義などのコード上の事実が記載されているか。
- [ ] **加点（Plussing）の存在**: 単なるバグ潰しに留まらず、アプリの世界観を活かした「ユーザーが嬉しくなる/ワクワクする」具体的な演出提案が含まれているか。
- [ ] **モチベーションの可視化**: 感情の谷（離脱ポイント）が明確に特定され、なぜそこで感情が下がるのかの心理的要因と、それを山に変える仮説が記述されているか。

## 判定
- 全✅ → 分析完了。`/ux-design` へ
- 未✅ → 該当STEPへ戻る
