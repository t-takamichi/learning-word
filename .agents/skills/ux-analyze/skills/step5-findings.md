# STEP 5: 課題統合・優先度付け・改善仮説

**目的**: STEP1〜4の指摘を統合し、次工程（/ux-design）が着手できる形に整理する。

## 実施内容
1. **課題の集約**: 重複を排除し、テーマ別（ビジュアル/モチベーション/サウンド/構造/アクセシビリティ）に分類
2. **優先度付け**: インパクト×実装コストの2軸でマトリクス化
   - 高インパクト×低コスト＝**最優先（Quick Win）**
3. **改善仮説**: 各課題に対し「こう変えれば、こう良くなる（と考えられる）」を1行で
4. **設計への引き継ぎ**: /ux-design で決めるべき論点をリスト化

## 出力フォーマット
```
## 課題サマリー（優先度順）
| # | テーマ | 課題（事実） | 改善仮説 | インパクト | コスト | 優先 |

## Quick Wins（最優先）
- ...

## 設計フェーズへの申し送り
- 決めるべき論点: 配色トークンの具体値 / 正解音の実装方式 / マスコット採否 ...
```
