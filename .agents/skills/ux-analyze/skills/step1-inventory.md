# STEP 1: 現状インベントリ（棚卸し）

**目的**: 分析対象の全体像を事実として把握する。

## 実施内容
- **画面一覧**: `src/client/pages/**` を確認し、画面と役割を列挙
- **コンポーネント一覧**: `src/client/components/**` を確認し、粒度（Atomic分類の見立て）を仮置き
- **デザイントークンの現状**: `index.css` と各 `*.module.css` から、使われている色・角丸・影・フォントを抽出
  - 特に **背景色・テキスト色・アクセント色** の実値（HEX/rgba）を表にする
- **フック/状態**: `src/client/hooks/**` を確認し、UIに関わる状態を把握
- **フィードバック要素の有無**: 音・アニメーション・祝福・褒め言葉が「今あるか」を確認（無ければ"無"と明記）

## 出力先
`docs/spec/design/ux/analysis/inventory.md`

## 出力フォーマット
- 画面表 / コンポーネント表 / 現状トークン表（色の実値付き）
- 「フィードバック機構の現状」チェックリスト（音/アニメ/祝福/マイクロコピー）
