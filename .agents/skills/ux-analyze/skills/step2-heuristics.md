# STEP 2: ヒューリスティック評価

**目的**: 確立された原則に照らし、UIの課題を減点法で機械的に洗い出す。

## 評価軸（Nielsen 10原則 ＋ モバイル観点）
1. システム状態の可視化（進捗・ローディングが分かるか）
2. 実世界との一致（言葉・比喩が対象ユーザーに馴染むか）
3. ユーザーの制御と自由（取り消し・戻れるか）
4. 一貫性と標準（画面間でトーンが揃うか）
5. エラー防止
6. 記憶より認識（覚えさせない）
7. 柔軟性と効率（片手・隙間時間で使えるか）
8. 美的でミニマルなデザイン
9. エラーからの回復
10. ヘルプとドキュメント

### モバイル追加観点
- タップターゲット44px / 本文16px / セーフエリア
- 片手操作での親指到達範囲
- iOS Safari 制約（音の解錠・vibrate非対応 等）

## 記法（各指摘）
```
[原則番号] 事実（どこがどうなっている）
  → なぜ問題か（誰がどう困る）
  → 深刻度（高/中/低）
  → 改善仮説（1行）
```
