# Quality Gate — UI/UX実装完了チェックリスト

以下を**全て**満たした場合のみ「UX-<N> 実装完了」とする。

## 実装
- [ ] 計画書（`docs/spec/impl/ux/phase-<N>.md`）の全タスク（T1〜Tn）を実装した
- [ ] 計画外の変更を勝手に加えていない（加えた場合はユーザー承認済み）
- [ ] Atomic Design の依存方向（下位は上位を import しない）に反していない
- [ ] 既存機能（出題ロジック・API・既存フック）を壊していない

## 品質・規約
- [ ] `tsc --noEmit`（型チェック）が通る（プロジェクトの `typecheck` スクリプト）
- [ ] `any` の乱用がなく、コンポーネントは `interface Props`（`React.FC` 不使用）
- [ ] 色・角丸・影が**トークン参照**で、`*.module.css` に生HEXが無い（`:root` と `#E5484D` を除く）
- [ ] 既存コードのスタイル・命名に揃っている

## アクセシビリティ / iOS
- [ ] 該当フェーズのアニメが `prefers-reduced-motion` で簡略化される
- [ ] サウンドを含む場合、AudioContext解錠・vibrate存在チェックが実装されている
- [ ] セーフエリア・タップ44px・本文16px を満たす

## 受け入れ基準
- [ ] 計画書の「受け入れ基準（AC）」チェックリストを全て満たす
- [ ] 対応する `docs/spec/design/ux/acceptance-criteria.md` の項目を満たす
- [ ] `phase-N.md` の「ゴール」が実際に動作確認できる

## 報告
- [ ] 実装内容と検証結果を正直に報告した（失敗・スキップ・iOS未検証も明示）

## 判定
- **全項目チェック済み** → UX-<N> 実装完了。次フェーズへ進む
- **未チェック項目あり** → 未完了。該当タスクへ戻って修正する
