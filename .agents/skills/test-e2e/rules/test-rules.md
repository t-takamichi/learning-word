# E2E Test Rules

1. `docs/test/test-plan.md` のE2Eシナリオ（ログイン → スタートボタン解錠 → カードフリップ → スワイプ/Good/Again → Undo → リザルト交代）を検証すること。
2. CSSハッシュ等に依存せず `getByRole` / `getByText` などのアクセシブルな要素特定を行うこと。
3. 非同期読み上げやトースト表示の要素待機を考慮し、Flaky にならないよう自動待機 (`toBeVisible()`) を使用すること。
4. モバイル・マルチデバイス表示時のレイアウト崩れや操作障害を検知すること。
