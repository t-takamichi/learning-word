# STEP 2: デザイントークン（Visual Design System）

**目的**: 色・タイポ・余白・角丸・影を **再利用可能な変数** として設計する。Atomic Designの"素粒子"層。

## 実施内容
- **カラーパレット**: ブランド/アクセント/ニュートラル/状態 を役割ごとにトークン化
  - 各色の **HEX実値** と用途を表で定義
  - コントラスト（本文はAA 4.5:1以上）を確認。ピンク上の白文字は要検証
  - 真っ黒を使わない（テキストはプラム系ニュートラル）
- **黒→新テーマ 置換対応表**: 現行CSSの生値 → 新トークン のマッピングを作る（移行を容易に）
- **タイポグラフィ**: 用途別サイズ/ウェイト/色。丸ゴシック系。本文最小16px
- **形/余白/影**: 角丸（card/btn/pill）・8ptグリッド・ソフト影トークン
- **マスコット方針**（採用する場合）: 最小コスト（絵文字/SVG）で導入、表情差分
- **`:root` 実装スニペット**: そのままCSSに落とせるトークン定義を記載

## 出力先
`docs/spec/design/ux/visual-design.md`

## 注意
- トークンは **命名で役割が分かる** こと（`--berry-500` 等、意味と階調）
- コンポーネント側に生HEXを書かせない前提で設計する
