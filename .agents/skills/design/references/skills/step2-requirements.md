# STEP 2: 要件整理（PM × エンジニア）

**目的**: スコープを確定し、優先順位をつける。

## 実施内容

### 機能要件（MoSCoW法で優先度付け）

| 優先度 | 意味 |
|--------|------|
| **Must have** | これがないとリリースできない |
| **Should have** | あるべきだが、なくてもリリース可能 |
| **Could have** | あると良い（余裕があれば） |
| **Won't have** | 今回はスコープ外（明示的に除外） |

### 非機能要件の定義
- パフォーマンス目標（TTI, FCP）
- 対応ブラウザ・デバイス（iPhone / Safari 優先）
- セキュリティ要件
- オフライン対応の有無

### スコープ外（Won't have）の明示
- スコープクリープを防ぐために「やらないこと」を明文化する
