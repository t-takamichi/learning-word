# STEP 5: リスク評価（PM）

**目的**: 設計段階で潜在リスクを可視化し、対策を検討する。

## 実施内容

### リスクマトリクス（影響度 × 発生確率）

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|----------|------|
| iOSのSpeechSynthesis制限 | 高 | 高 | ユーザーアクション起点に限定 |
| SQLiteの同時書き込み | 中 | 低 | WALモード有効化 |
| Safari非対応API | 高 | 中 | PoC検証を先行実施 |
| スコープクリープ | 中 | 高 | Won't have を明示してロック |
| （追加リスク） | - | - | - |

### リスクレベルの定義
- **高**: プロジェクトの成功に重大な影響を与える
- **中**: 対処しないと品質・スケジュールに影響する
- **低**: 許容範囲、モニタリング継続

### 技術的スパイク（事前検証）が必要な項目
- リスクが「高×高」の項目は実装前にPoCを行う
