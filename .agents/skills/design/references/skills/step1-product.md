# STEP 1: プロダクト背景の整理（PM）

**目的**: 設計を始める前に「なぜ作るか」を言語化する。

## 実施内容

### ユーザーストーリーの作成
```
As a [ユーザー],
I want to [機能・行動],
So that [得られる価値・目的].
```

### ターゲットユーザー（ペルソナ）の定義
- 誰が使うか
- どんな場面で使うか
- 技術リテラシーレベル

### 解決するペインポイントの明示
- 現状の課題を箇条書きで列挙
- 「このアプリがないと何が困るか」

### 成功指標（KPI）の仮定義
- 測定可能な指標を最低1つ設定する
- 例: DAU、継続率、タスク完了率、セッション時間
