## 構造
- [ ] Atomic Design のボトムアップ順でフェーズが並んでいる
- [ ] フェーズ依存が矛盾していない（下位→上位）
- [ ] リスクの高いサウンド（iOS解錠）が早いフェーズにある

## タスク
- [ ] 各タスクが「成果物・内容・層・検証方法・適用ルール」を持つ
- [ ] 各タスクが検証可能な粒度（1関心事）
- [ ] 生HEX禁止・トークン参照が徹底される計画になっている
- [ ] useSound（AudioContext解錠・vibrate存在チェック・ミュート保持）がタスク化されている
- [ ] 正解演出の結線（Goodで音×視覚×言葉を200ms以内）がタスク化されている

## AC / スコープ
- [ ] 各フェーズに受け入れ基準がある（UX仕様のACと対応）
- [ ] iOS Safari実機・reduced-motion・コントラストの検証手順がある
- [ ] Could（スコープ外）が混入していない
- [ ] 機能ロジックを壊す計画になっていない

## 判定
- 全✅ → 実装計画完了。`/impl <フェーズ番号>` へ
- 未✅ → 該当STEPへ戻る
