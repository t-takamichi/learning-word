# Quality Gate — 設計完了チェックリスト

以下を**全て**満たした場合のみ「設計完了」とし、実装フェーズ（impl）へ進む。

## ドキュメント
- [ ] `docs/spec/design/README.md` が存在し、全体概要が書かれている
- [ ] `docs/spec/design/architecture.md` が存在し、技術選定の根拠が書かれている
- [ ] 各機能の詳細設計ファイル `docs/spec/design/[feature].md` が存在する
- [ ] `docs/spec/design/risks.md` が存在する

## PM観点
- [ ] 全機能にユーザーストーリーが書かれている
- [ ] 全機能にMoSCoW優先度が付いている
- [ ] Must have の全機能に受け入れ基準（AC）が書かれている
- [ ] 成功指標（KPI）が少なくとも1つ定義されている
- [ ] Won't have（スコープ外）が明示されている

## エンジニア観点
- [ ] シーケンス図（Mermaid）が主要フローに存在する
- [ ] データモデル（ERD）が定義されている
- [ ] APIエンドポイント一覧が定義されている
- [ ] リスク「高×高」の項目に対策が書かれている

## 判定
- **全項目チェック済み** → 設計完了。`docs/spec/impl/` フェーズへ進む
- **未チェック項目あり** → 設計未完了。該当STEPに戻って補完する
