# STEP 1: デザインコンセプト・原則・ムード

**目的**: 全設計判断の拠り所となる"世界観"を先に固める。

## 実施内容
- **コンセプトワード**: このプロダクトの体験を一言で（例:「がんばりが、ごほうびになる単語帳」）
- **ブランドムード（3語）**: 例「やさしい／あまい／前向き」
- **トーン&マナー**: 形・色・言葉の方向性（例: 丸み・淡いピンク・応援形の言葉・真っ黒を使わない）
- **デザイン原則（P1〜P5）**: 迷ったときの判断順序を定義
  - Delight優先 / 頑張りを見逃さない / 片手3秒 / うるさくしない / 細部にやさしさ
- **スコープ**: In / Out を明示（機能ロジックは変えない等）

## 入力
- 分析の申し送り（`analysis/findings.md`）
- ユーザー要望（ピンク基調・モチベ・正解音・ディズニー品質）

`docs/spec/design/ux/README.md`（UX仕様の入口。以降のファイルへリンク）
