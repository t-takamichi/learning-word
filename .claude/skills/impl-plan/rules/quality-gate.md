# Quality Gate — 実装計画完了チェックリスト

以下を**全て**満たした場合のみ「実装計画完了」とし、実装フェーズ（impl）へ進む。

## ドキュメント
- [ ] `docs/spec/impl/steps/<フェーズ番号>/plan.md` が存在する
- [ ] 計画書が対応する `phase-N.md` と設計書へリンクされている

## トレーサビリティ
- [ ] フェーズ定義のスコープが全てタスクへ落とし込まれている
- [ ] フェーズのスコープ外の作業が計画に混入していない
- [ ] 設計書のAC（受け入れ基準）がDoneチェックリストへ転記されている

## タスク品質
- [ ] 各タスクに連番（T1, T2, ...）が振られ、実装順が一意である
- [ ] タスク順序がレイヤー依存方向（型→Domain→Infra→App→Presentation→Frontend）に沿う
- [ ] 各タスクに対象ファイルパスが記載されている
- [ ] 各タスクに検証方法（動作確認/テスト）が記載されている
- [ ] 適用される `docs/rules/` のルールが紐づけられている

## 判定
- **全項目チェック済み** → 実装計画完了。`/impl <フェーズ番号>` へ進む
- **未チェック項目あり** → 未完了。該当STEPに戻って補完する
