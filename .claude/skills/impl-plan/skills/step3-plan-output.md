# STEP 3: 実装計画書の生成

**目的**: 分解したタスク列を実装計画書として書き出し、実装フェーズ（impl）へ引き渡す。

## 保存先ディレクトリ構成

```
docs/spec/
└── impl/
    └── steps/
        └── <フェーズ番号>/
            └── plan.md      ← 実装計画書
```

- 例: Phase 1 → `docs/spec/impl/steps/1/plan.md`

## plan.md の必須構成

```markdown
# Phase <N> 実装計画書: [フェーズ名]

## 1. 参照
- フェーズ定義: [phase-N.md](../../phase/phase-N.md)
- 設計: [feature.md](../../../design/[feature].md)

## 2. このフェーズのゴール
- phase-N.md の「ゴール」を転記

## 3. 前提・依存
- 前提となるフェーズ / 事前準備（環境・依存パッケージ）

## 4. 実装タスク一覧（実装順）
| ID | 成果物（ファイル） | 内容 | 層 | 検証方法 | 適用ルール |
|----|------------------|------|----|---------|-----------|
| T1 | ... | ... | ... | ... | ... |
| T2 | ... | ... | ... | ... | ... |

## 5. 各タスク詳細
### T1: [タイトル]
- **ファイル**: `path/to/file.ts`（新規/変更）
- **やること**: 主要な型・関数シグネチャ・処理概要
- **検証**: 動作確認手順
- **注意**: 適用ルール・落とし穴

（T2 以降も同様に記述）

## 6. Done の定義（受け入れ基準チェックリスト）
- [ ] 設計書のAC-01 を満たす
- [ ] 設計書のAC-02 を満たす
- [ ] `tsc --noEmit` が通る
- [ ] フェーズのゴールが動作確認できる
```

## 完了条件
- `docs/spec/impl/steps/<番号>/plan.md` が存在すること
- `rules/quality-gate.md` の全チェック項目がパスすること

## 完了後の案内
```
Phase <N> 実装計画完了。次は計画に沿って実装を開始します:
/impl <N>
```
