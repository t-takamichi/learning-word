# STEP 1: フェーズ定義・設計の読み込み

**目的**: 実装計画の材料として、対象フェーズの範囲と設計詳細を把握する。

## 実施内容

### 対象フェーズの特定
- 引数のフェーズ番号 `N` を確認する
- `docs/spec/impl/phase/phase-N.md` を読み込む
  - ゴール / スコープ / スコープ外 / 依存フェーズ / 完了条件 / リスク

### 依存フェーズの確認
- `phase-N.md` の「依存フェーズ」を確認し、前提フェーズが計画済み/実装済みかを確認する
- 未完了の前提があれば、それを先に処理するよう案内する

### 設計書との突き合わせ
- フェーズが参照する設計詳細を読み込む
  ```
  docs/spec/design/[feature].md   ← シーケンス図・AC・データモデル
  docs/spec/design/architecture.md ← APIエンドポイント・ディレクトリ構成
  ```
- 実装に必要な型・API・画面・DBスキーマを具体的に把握する

### 実装ルールの確認
- `docs/rules/` の該当ルール（TypeScript / Hono / React / architecture）を確認する

## 出力
このSTEPは内部整理のみ（ファイル出力なし）。
把握した内容を次STEPのタスク分解へ引き継ぐ。
