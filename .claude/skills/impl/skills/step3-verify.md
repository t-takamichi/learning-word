# STEP 3: 検証・受け入れ基準の確認・報告

**目的**: フェーズ全体が計画どおり動くことを確認し、正直に報告する。

## 実施内容

### 全体ビルド・型チェック
- プロジェクト全体で型チェック・ビルドを実行する
  ```
  tsc --noEmit
  （ビルド/テストのスクリプトがあれば実行）
  ```

### Done の定義を確認
- 計画書 `plan.md` の「Done の定義（受け入れ基準チェックリスト）」を上から確認する
- 各項目に実際にチェックを入れられるか、動作で確認する
- 対応する設計書のAC（受け入れ基準）を満たしているかも突き合わせる

### フェーズゴールの動作確認
- `phase-N.md` の「ゴール（このフェーズ完了時に動くもの）」を実際に動かして確認する
- 必要に応じて `/run`（アプリ起動）や `/verify`（挙動確認）を活用する

### 正直な報告
実装結果をユーザーへ報告する。以下を必ず含める。

- 実装したタスク（T1〜Tn）と、それぞれの検証結果
- 受け入れ基準チェックリストの充足状況（満たした / 未達）
- 落ちたテスト・スキップした確認があれば、その内容を明示
- 「完了」と言えるのは検証が通ったときのみ

## 完了条件
- `rules/quality-gate.md` の全チェック項目がパスすること

## 完了後の案内
```
Phase <N> 実装完了。次のフェーズへ進む場合:
/impl-plan <次のフェーズ番号>  →  /impl <次のフェーズ番号>
```
