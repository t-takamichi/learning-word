# Impl Rules

実装時に必ず守るルール。違反する実装は受け付けない。

---

## 前提ルール

### R-IM-PRE-01: 実装前に docs/rules を必ず読む
- `docs/rules/architecture.md` `TypeScript.md` `hono.md` `react.md` を実装前に確認する
- これらの規約に反する実装をしない

### R-IM-PRE-02: 計画書に沿って実装する
- `docs/spec/impl/steps/<番号>/plan.md` のタスク（T1→Tn）を順序どおりに実装する
- 計画に存在しない作業を勝手に追加しない（必要ならユーザーへ確認）

---

## 実装ルール

### R-IM-01: レイヤー依存方向を守る
- `Presentation → Application → Domain ← Infrastructure` の依存方向を厳守する
- Application 層に Hono の `Context` を持ち込まない、等の禁止事項を守る

### R-IM-02: 型安全を徹底する
- `any` を安易に使わない
- Hono API は `zValidator` でスキーマ検証する
- フロント・バックで共有型（`shared/`）を使う

### R-IM-03: 1タスクごとに検証する
- 各タスク完了時に、計画書に書かれた検証方法を実行する
- 検証が通ってから次のタスクへ進む

### R-IM-04: 既存コードのスタイルに合わせる
- 周囲の命名・フォーマット・コメント密度に合わせる
- 無関係なリファクタリングを混ぜない

### R-IM-05: 破壊的操作は確認してから行う
- 既存ファイルの大幅な上書き・削除は、内容を確認してから行う
- マイグレーションでの破壊的変更は影響を明示する

---

## 報告ルール

### R-IM-REP-01: 結果を正直に報告する
- テストが落ちたら、出力とともに「落ちた」と報告する
- スキップした手順があれば明示する
- 「完了」と言えるのは検証が通ったときだけ
